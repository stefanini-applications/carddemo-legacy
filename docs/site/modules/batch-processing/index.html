⚙️ Batch Processing

User Story Development Guide — CardDemo Application

📋 Overview

Purpose: Execute nightly and periodic batch jobs: transaction posting, interest calculation, statement generation, account/customer file operations, and data export/import.

Business Context: Batch processing is the backbone of CardDemo's financial operations. Daily transactions must be posted, interest calculated monthly, and statements generated for billing. These jobs run outside business hours via JCL.

Batch COBOL JCL VSAM Nightly + Monthly

🎯 User Story Development Guide

📝 Module-Specific US Templates

  • Daily Posting: As a batch operator, I want daily transactions posted nightly so that account balances are current by the next business day.
  • Interest Calculation: As a batch operator, I want interest calculated monthly so that finance charges appear on cardholder statements.
  • Statement Generation: As a batch operator, I want statements generated for all active accounts so that cardholders receive their monthly bills.
  • Reject Handling: As a batch operator, I want rejected transactions captured in a reject file so that exceptions can be reviewed and resolved.
  • Data Export: As an integration team, I want account/transaction data exported so that it can be consumed by downstream systems.

🔗 VSAM Dataset Dependencies

  • DALYTRAN: Daily transaction input (sequential) → fed to CBTRN01C/02C
  • TRANSACT: Transaction master (KSDS) → written by posting, read by reports
  • ACCTDAT: Account master → balance updated by posting, interest calculation
  • TCATBALF: Transaction category balance → updated by posting, read by interest calc
  • DISCGRP: Discount group interest rates → read by CBACT04C
  • DALYREJS: Daily reject file (sequential) → written by CBTRN02C for exceptions

📊 Interest Calculation Logic

For each TCATBALF record (Account + Type + Category):
  Look up interest rate in DISCGRP:
    Key = ACCT-GROUP-ID + TRAN-TYPE-CD
  Calculate: Interest = TRAN-CAT-BAL * (Rate / 12)
  Create interest TRAN-RECORD in TRANSACT
  Update ACCTDAT: ACCT-CURR-BAL += Interest
  Update TCATBALF balance

✅ Acceptance Criteria Patterns

Daily Transaction Posting:
  • Given DALYTRAN contains N transaction records
  • When CBTRN01C/02C runs successfully
  • Then all valid records posted to TRANSACT; ACCTDAT balances updated; TCATBALF updated; reject file contains only invalid records
  • And job prints control totals (records read, posted, rejected)
Interest Calculation:
  • Given TCATBALF contains balances and DISCGRP has rates
  • When CBACT04C runs
  • Then interest transactions created for all non-zero balances; ACCTDAT balances updated
