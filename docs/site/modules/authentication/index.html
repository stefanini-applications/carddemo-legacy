🔐 Authentication

User Story Development Guide — CardDemo Mainframe Application

Overview

Controls application entry via the CC00 CICS transaction. Reads the USRSEC VSAM security file to validate credentials. Routes admin users (USER-TYPE=A) to the Admin Menu (COADM01C) and regular users (USER-TYPE=U) to the Main Menu (COMEN01C).

Technical Foundation

Programs

  • COSGN00C — Signon logic: validates USER-ID/PASSWORD against USRSEC VSAM
  • COMEN01C — Main Menu: entry point for regular users
  • COADM01C — Admin Menu: entry point for admin users

BMS Maps

  • COSGN00 — Signon screen map
  • COMEN01 — Main menu map
  • COADM01 — Admin menu map

User Story Development Guide

Story Templates

Security Officer As a security officer, I want failed login attempts to display an error message so that unauthorized access is prevented. Security

Writing Good Stories for This Module

  • Reference the CICS transaction (CC00) and the routing program (COSGN00C) in technical notes.
  • Specify the VSAM file (USRSEC) and key field (USER-ID) when stories involve credential data.
  • Always distinguish between USER-TYPE=A and USER-TYPE=U routing outcomes.
  • Use the commarea copybook COCOM01Y reference when stories affect session state.

Complexity Grid

🟢 Simple (1–2 pts)

  • Password change (USRSEC update only)
  • Login screen cosmetic change (BMS map update)
  • Add descriptive error text to signon screen

🟡 Medium (3–5 pts)

  • Add account lockout after N failed login attempts
  • Add new user role beyond A/U (requires routing logic change)
  • Session timeout with configurable inactivity period

🔴 Complex (5–8 pts)

  • Integrate with RACF passtickets for SSO
  • SSO integration via MQ message-based token exchange
  • Audit trail of all login events to a new VSAM or DB2 table

Business Rules

  • Routing: USER-TYPE=A → COADM01C (Admin Menu); USER-TYPE=U → COMEN01C (Main Menu).
  • Failed auth: Stays on the signon screen; no partial session is created.
  • Session state: CICS commarea (COCOM01Y) carries session context between programs throughout the user session.
  • Credentials: USER-ID is 8 characters, PASSWORD is 8 characters; both stored in USRSEC. Password is currently stored as plain text (modernization target).
  • VSAM record: USRSEC is 80-byte fixed-block; keyed on USER-ID (first 8 bytes).
