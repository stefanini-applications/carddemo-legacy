📋 Module Overview

Purpose: Maintain transaction type reference data in IBM DB2 using CICS online screens and batch programs. Demonstrates DB2 integration patterns including static embedded SQL, cursors, and SQLCA error handling.

Business Context: Transaction types categorize every financial transaction (purchase, refund, cash advance, etc.). This module allows administrators to maintain this reference data directly in DB2 instead of just flat VSAM files, enabling richer reporting and more flexible type management.

Transactions: CTTU (add/edit) | CTLI (list/delete)
Programs: COTRTUPC / COTRTLIC / COBTUPDT (batch)

COBOL CICS IBM DB2 Static SQL Cursors Optional Module

🎯 User Story Templates

US-TTDB2-001: As an administrator, I want to list transaction types from DB2 so that I can review the current reference data.
US-TTDB2-002: As an administrator, I want to add a new transaction type to DB2 so that new merchant categories can be processed.
US-TTDB2-003: As an administrator, I want to update an existing transaction type description so that the reference data stays accurate.
US-TTDB2-004: As an administrator, I want to delete an obsolete transaction type from DB2 so that invalid codes cannot be used.

✅ Acceptance Criteria Patterns

Add Transaction Type:
GIVEN a unique transaction type code and description
WHEN CTTU is submitted
THEN a DB2 INSERT is executed, SQLCODE = 0, and confirmation is displayed
Duplicate Code:
GIVEN a transaction type code that already exists in DB2
WHEN CTTU is submitted
THEN SQLCODE -803 (duplicate key) is handled and an error message is displayed
List with Pagination:
GIVEN transaction types exist in DB2
WHEN CTLI is executed
THEN types are listed using a DB2 cursor; PF7/PF8 navigate forward/backward
TRANEXTR Batch:
GIVEN DB2 transaction type table has current data
WHEN TRANEXTR runs DSNTIAUL
THEN VSAM TRANTYPE file is updated with DB2 data, return code = 0

DB2 Integration Patterns Demonstrated

  • Static embedded SQL: SQL statements precompiled into the program (COTRTUPC uses INSERT, UPDATE)
  • Forward cursor: COTRTLIC uses DECLARE CURSOR / OPEN / FETCH / CLOSE for sequential browsing
  • Backward cursor: COTRTLIC also demonstrates reverse cursor scrolling (PF7)
  • SQLCA error handling: All SQL operations check SQLCODE; specific codes handled (e.g., -803 duplicate, +100 not found)
  • DELETE with cursor: COTRTLIC demonstrates positioned DELETE via cursor
  • DB2 precompiler: Programs use EXEC SQL statements compiled with DB2 precompiler before COBOL compilation

Integration with Base Application

  • TRANEXTR extracts DB2 transaction type data to VSAM TRANTYPE file
  • Base application reads TRANTYPE VSAM for transaction validation
  • Admin Menu (CA00) options 5 and 6 activate CTTU and CTLI transactions
  • CREADB21 must run before first use to create DB2 tables

Prerequisites

  • Base CardDemo installed and operational
  • IBM DB2 configured and accessible from CICS
  • CREADB21 JCL executed to create DB2 database and tables
  • CICS resources defined for CTTU and CTLI transactions
  • DB2 precompiler available for program compilation

⚡ Development Acceleration Factors

  • COTRTUPC INSERT/UPDATE pattern: Reusable static embedded SQL pattern for DB2 CICS programs
  • COTRTLIC cursor pattern: Forward/backward cursor scrolling reference implementation
  • SQLCA error handling: Established error handling framework for DB2 SQLCODE checking
  • TRANEXTR pattern: DB2-to-VSAM extraction pattern using DSNTIAUL - reuse for other DB2 extractions
