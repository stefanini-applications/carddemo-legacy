📋 Module Overview

Purpose: Manages credit card records – listing cards for an account, viewing card details, and updating card information.

Business Context: Cardholders can see all cards linked to their account, review individual card details (number, expiry, status), and request updates. Customer service representatives can activate, block, or update card information.

Transactions: CCLI (list) | CCDL (view) | CCUP (update)
Programs: COCRDLIC / COCRDSLC / COCRDUPC

COBOL CICS VSAM CARDFILE VSAM CARDXREF BMS Maps

🎯 User Story Templates

US-CARD-001: As a cardholder, I want to list all my credit cards so that I can see which cards are linked to my account.
US-CARD-002: As a cardholder, I want to view a card's details (number, expiry, status) so that I can verify my card information.
US-CARD-003: As a customer service representative, I want to update a card's active status so that I can activate or block a card as needed.
US-CARD-004: As a cardholder, I want to see the name embossed on my card so that I can confirm the card is correctly personalized.
US-CARD-005: As a customer service representative, I want to update a card's expiration date so that I can issue a renewed card.

✅ Acceptance Criteria Patterns

Card List:
GIVEN an authenticated user with account ID
WHEN CCLI transaction is executed
THEN all cards linked to the account are listed with card number and status
Card View:
GIVEN a valid card number selected from the list
WHEN CCDL transaction is executed
THEN all card fields are displayed (number, CVV masked, expiry, embossed name, status)
Card Update – Status:
GIVEN a CSR updates card active status to N (blocked)
WHEN the update is submitted
THEN CARDFILE is updated and subsequent transactions on that card are declined

Data Model – CARD-RECORD (CVACT02Y)

01 CARD-RECORD.
   05 CARD-NUM               PIC X(16)    -- 16-digit card number (PAN key)
   05 CARD-ACCT-ID           PIC 9(11)    -- Associated account ID
   05 CARD-CVV-CD            PIC 9(03)    -- CVV security code (3 digits)
   05 CARD-EMBOSSED-NAME     PIC X(50)    -- Name embossed on card
   05 CARD-EXPIRAION-DATE    PIC X(10)    -- Expiry date (YYYY-MM-DD)
   05 CARD-ACTIVE-STATUS     PIC X(01)    -- Y=Active, N=Inactive/Blocked
   05 FILLER                 PIC X(59)

Dependencies

  • authentication: Session required before accessing card functions
  • accounts: CARD-ACCT-ID links cards to ACCTFILE records
  • CARDFILE/CARDXREF VSAM: Loaded via CARDFILE and XREFFILE batch jobs
  • authorization (optional): Card status affects authorization decisions

⚡ Development Acceleration Factors

  • CVACT02Y copybook: Standard card record – reuse for all card operations
  • CVACT03Y cross-reference: Use CARDXREF to navigate Account ↔ Card ↔ Customer relationships
  • COCRDLIC list pattern: PF key pagination pattern (PF7/PF8) established – reuse for other list screens
