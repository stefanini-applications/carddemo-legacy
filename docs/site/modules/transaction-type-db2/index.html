📋 Transaction Type DB2 Module

CardDemo — User Story Development Guide

📋 Overview

Purpose: Manage transaction type and transaction category reference data stored in DB2 tables, accessible through admin CICS transactions and batch maintenance jobs.

Business Context: Transaction types (2-char codes) and categories (4-digit codes) classify card spending for interest calculation and reporting. This module stores them in DB2 and provides admin CICS screens and batch ETL to synchronize with VSAM for online use.

CICS Transactions: CTTU (add/edit type), CTLI (list/update/delete) — Admin Menu options 5 and 6

🎯 User Story Development Guide

📝 US Templates

  • As an admin, I want to add a new transaction type to DB2 so that new card product categories are supported
  • As an admin, I want to list all transaction types from DB2 so that I can review current reference data
  • As an admin, I want to update a transaction type description so that reporting reflects current business terminology
  • As an admin, I want to delete an obsolete transaction type from DB2 so that it no longer appears in reports
  • As a batch operator, I want to run TRANEXTR to export DB2 transaction types to VSAM so that online programs have current data

⚡ Development Acceleration Factors

  • DB2 cursor pattern: COTRTLIC demonstrates DB2 cursor, update, and delete — reference for other DB2 programs
  • DB2 insert/update pattern: COTRTUPC demonstrates DB2 INSERT and UPDATE via embedded SQL
  • DCL files: DCLTRCAT.dcl, DCLTRTYP.dcl — DB2 table declarations for copybook generation
  • TRANEXTR pattern: DB2-to-VSAM extract (DSNTIAUL) reusable for other DB2 export needs

📊 Complexity Guidelines

  • 3–5 pts Tran Type List (CTLI) — DB2 cursor scan, display, delete
  • 3–5 pts Tran Type Add/Edit (CTTU) — DB2 INSERT or UPDATE with validation
  • 5–8 pts TRANEXTR batch + MNTTRDB2 — DB2 unload to VSAM, batch table maintenance

📊 DB2 Tables

TRNTYPE:
  TRAN-TYPE-CD    CHAR(02)  -- Primary key: 2-char type code (e.g., 'PR', 'SA')
  TRAN-TYPE-DESC  VARCHAR   -- Description

TRNTYCAT:
  TRAN-TYPE-CD    CHAR(02)  -- FK to TRNTYPE
  TRAN-CAT-CD     CHAR(04)  -- 4-digit category code
  TRAN-CAT-DESC   VARCHAR   -- Category description

✅ Acceptance Criteria Examples

  • Given admin menu option 5 or 6 → CTTU/CTLI accessible only when this DB2 module is installed
  • Given valid new type code on CTTU → DB2 INSERT succeeds, new type visible in CTLI list
  • Given duplicate type code on CTTU → DB2 constraint violation handled, error message shown
  • Given CTLI list → All DB2 transaction types displayed via cursor scan
  • Given delete on CTLI → DB2 DELETE executed, type removed from list
  • Given TRANEXTR batch runs → Latest DB2 transaction type data exported to flat file and loaded to VSAM for online use
