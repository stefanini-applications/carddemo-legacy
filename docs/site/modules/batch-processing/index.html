📋 Overview

The Batch Processing module defines the end-of-day (EOD) and periodic JCL pipeline that processes transactions, calculates interest, updates accounts, generates statements, and manages VSAM files. All online CICS activity pauses while batch runs (CLOSEFIL), and resumes when batch completes (OPENFIL).

Business Context

The batch pipeline runs after CICS is quiesced using the CLOSEFIL job to prevent concurrent VSAM updates. The full EOD sequence is orchestrated by JCL jobs in app/jcl/. Scheduler definitions are provided for both CA7 and Control-M in app/scheduler/. GDG datasets retain statement history across runs. The pipeline concludes with OPENFIL to restore CICS access.

🎯 User Story Development Guide

EOD Job Sequence

  1. CLOSEFIL — quiesce CICS, close VSAM files for exclusive batch access
  2. TRANBKP — backup TRANSACT VSAM before posting
  3. POSTTRAN (CBTRN02C) — post DALYTRAN records to TRANSACT VSAM; update ACCTDATA balances
  4. INTCALC (CBACT04C) — calculate interest on all account balances using DISCGRP rates
  5. COMBTRAN — combine transaction files for downstream processing
  6. CREASTMT (CBSTM03A/B) — generate account statements, write to GDG
  7. TRANIDX — rebuild TRANSACT AIX for online card-number-based lookup
  8. OPENFIL — restore CICS access to VSAM files

Story Templates

Run Nightly Batch to Post Daily Transactions
As a batch operations manager, I want POSTTRAN to post all DALYTRAN records to TRANSACT VSAM each night so that cardholder balances are current by morning.

CBTRN02C reads DALYTRAN sequentially, inserts each record to TRANSACT VSAM KSDS, updates ACCTDATA balance; RC=0 required
Calculate Interest on Outstanding Balances
As a finance manager, I want INTCALC (CBACT04C) to apply interest charges to all accounts each night so that finance charges are accurate.

CBACT04C reads ACCTDATA sequentially, looks up ACCT-GROUP-ID in DISCGRP, calculates interest, updates TCATBALF and ACCTDATA balance
Generate Monthly Statements
As a billing administrator, I want CREASTMT to generate a statement for every active account at end of billing cycle so that statements are available for mailing.

CBSTM03A reads ACCTDATA + TRANSACT VSAM, formats statement, writes to GDG (+1 generation); CBSTM03B provides support routines
Restart Failed Batch from Checkpoint
As a batch operations manager, I want to restart a failed batch job from its last checkpoint so that I do not have to reprocess already-completed records.

Requires checkpoint/restart implementation (Complex) — CBTRN02C position save + POSTTRAN JCL RESTART parameter
Schedule Batch via CA7 / Control-M
As a scheduling administrator, I want to define the EOD job dependencies in the scheduler so that jobs run automatically in the correct sequence each night.

app/scheduler/CardDemo.ca7 (CA7) and app/scheduler/CardDemo.controlm (Control-M) provide ready-to-use scheduler definitions

Acceleration Factors

  • All JCL jobs provided — complete EOD pipeline JCL in app/jcl/; new stories extend existing jobs
  • CBTRN02C sequential-to-KSDS post pattern — the template for any new batch posting story
  • CBACT04C interest rate lookup pattern — DISCGRP read + rate application is reusable for any discount/rate calculation story
  • IDCAMS for VSAM operations — DEFINE/DELETE/REPRO/BLDINDEX patterns provided in existing JCL
  • GDG patterns in CREASTMT — GDG (+1) dataset allocation and write pattern reusable for any archival output story

✅ Acceptance Criteria Patterns

Transaction Posting (POSTTRAN)

  • GIVEN a DALYTRAN file with valid records, WHEN POSTTRAN runs, THEN all records are posted to TRANSACT VSAM KSDS with RC=0
  • GIVEN posted transactions, WHEN POSTTRAN completes, THEN account balances in ACCTDATA VSAM reflect all posted transaction amounts

Interest Calculation (INTCALC)

  • GIVEN accounts with varying group IDs, WHEN INTCALC runs, THEN each account's interest is calculated using the correct rate from DISCGRP for its group ID
  • GIVEN a zero-balance account, WHEN INTCALC processes it, THEN no interest charge is applied

Statement Generation (CREASTMT)

  • GIVEN a CREASTMT run, WHEN CBSTM03A completes with RC=0, THEN the GDG generation number is incremented and each active account has a statement

File Management

  • GIVEN CLOSEFIL has run, WHEN CICS attempts to access a managed VSAM file, THEN the access is refused until OPENFIL completes
  • GIVEN the full EOD pipeline completes, WHEN OPENFIL runs, THEN all CICS-managed VSAM files are accessible to online transactions
