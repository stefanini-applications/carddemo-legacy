💳 Credit Cards

User Story Development Guide — CardDemo Mainframe Application

Overview

Supports listing (CCLI), viewing (CCDL), and updating (CCUP) credit card records. Cards are stored in the CARDFILE VSAM KSDS (150-byte records). Card-to-account-to-customer cross-referencing is handled by CARDXREF VSAM with an Alternate Index (AIX). Multiple cards may belong to one account.

User Story Development Guide

Writing Good Stories for This Module

  • Always specify which VSAM file is involved: CARDFILE for card data, CARDXREF for cross-reference lookups.
  • Deactivation stories must include criteria for what happens to in-flight transactions.
  • Expiration and reissue stories span both CARDFILE and CARDXREF — note both files in technical notes.
  • New card issuance must update both CARDFILE and CARDXREF atomically.

Complexity Grid

🟢 Simple (1–2 pts)

  • View card list for an account (read-only)
  • View card detail including expiry and status
  • Display card number masked (last 4 digits)

🟡 Medium (3–5 pts)

  • Update card active status with validation and VSAM rewrite
  • Update embossed name with format validation
  • Filter card list to show only active cards

🔴 Complex (5–8 pts)

  • Card reissue: update expiry + CVV + CARDXREF cross-reference atomically
  • New card issuance: write CARDFILE + CARDXREF records with unique card number generation
  • Card replacement workflow with status history audit trail

Acceptance Criteria

A card expiry date has passed The card detail (CCDL) is viewed Expiry status is shown and renewal prompt is displayed

Business Rules

  • Cross-reference: CARDXREF links card number → account ID → customer ID. All three must remain consistent.
  • Expiry enforcement: Card expiration date is checked during authorization; expired cards are declined.
  • Active status: CARD-ACTIVE-STATUS=Y is required for transaction eligibility; setting to N immediately blocks new transactions.
  • Multiple cards: One account may have multiple active cards; each is an independent CARDFILE record.
  • Record size: CARDFILE is 150-byte fixed-block KSDS; keyed on CARD-NUM (16 chars).
