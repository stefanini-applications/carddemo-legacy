📊 Reports Module

CardDemo — User Story Development Guide

📋 Overview

Purpose: Provides online report request capability and batch statement/report generation for credit card accounts.

Business Context: Cardholders can trigger transaction reports online (CORPT00C submits JCL via extra-partition TDQ). Batch generates account statements in plain text and HTML formats. The CBSTM03A program demonstrates advanced COBOL patterns used in modernization scenarios.

CICS Transaction: CR00 (report request online) | Batch Jobs: CREASTMT, TRANREPT

🎯 User Story Development Guide

📝 US Templates

  • As a cardholder, I want to request a transaction report online so that I can receive a statement without calling support
  • As a batch operator, I want to run CREASTMT so that all cardholders receive their monthly statements in text and HTML
  • As a financial auditor, I want transaction reports with full detail so that I can verify all charges
  • As a modernization engineer, I want to understand CBSTM03A's COBOL patterns (ALTER/GOTO, arrays, subroutines) so that I can safely transform this program

⚡ Development Acceleration Factors

  • CORPT00C TDQ pattern: Online-to-batch JCL submission via extra-partition TDQ already implemented
  • CBSTM03B subroutine: Reusable statement formatting logic called by CBSTM03A
  • TRANFILE AIX on date: Date-range query capability enables scoped reports

📊 Complexity Guidelines

  • 1–2 pts Online report request screen (CR00) — display form and submit JCL
  • 3–5 pts Batch transaction report (TRANREPT) — date-range query and formatted output
  • 5–8 pts Account statement (CREASTMT/CBSTM03A) — multi-account, dual format (text + HTML), subroutine

✅ Acceptance Criteria Examples

  • Given authenticated user on CR00 → Report request screen displays date range input fields
  • Given date range submitted → JCL submitted to TRANREPT job via internal reader TDQ
  • Given CREASTMT batch runs → Each account has a text statement and an HTML statement file
  • Given account statement → All transactions for the cycle are present, totals match ACCTFILE balances
  • Given TRANREPT batch → Report covers only transactions within specified date range using AIX
