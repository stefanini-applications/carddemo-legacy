💳 Credit Cards Module

CardDemo — User Story Development Guide

📋 Overview

Purpose: Provides cardholders with the ability to list, view details, and update credit cards linked to their account.

Business Context: Each account can have one or more credit cards. The XREFFILE maintains the bi-directional relationship between cards, accounts, and customers. Card status controls whether transactions can be processed against it.

CICS Transactions: CCLI (list), CCDL (view), CCUP (update)

🎯 User Story Development Guide

📝 US Templates

  • As a cardholder, I want to see all credit cards linked to my account so that I can manage them
  • As a cardholder, I want to view the details of a specific card (number, expiry, status) so that I can verify its validity
  • As a cardholder, I want to update the embossed name on my card so that it reflects my current name
  • As an account manager, I want to deactivate a card (set status to inactive) so that a lost or stolen card cannot be used
  • As a cardholder, I want to see my card expiration date so that I can request a replacement before it expires

⚡ Development Acceleration Factors

  • CVACT02Y copybook: CARD-RECORD structure (150-byte) pre-defined
  • XREFFILE pattern: Card-to-account lookup via XREFFILE alternate index already implemented in transaction module
  • BMS maps: COCRDLI, COCRDSL, COCRDUP all pre-defined

📊 Complexity Guidelines

  • 1–2 pts Card List (CCLI), Card View (CCDL) — read from CARDFILE/XREFFILE
  • 3–5 pts Card Update (CCUP) — validate and rewrite CARDFILE; activate/deactivate
  • 5–8 pts New card issuance with XREFFILE linkage; multi-card account management

📊 Card Data Model

CARD-NUM              PIC X(16)  -- Primary key (16-digit card number)
CARD-ACCT-ID          PIC 9(11)  -- Linked account
CARD-CVV-CD           PIC 9(03)  -- Verification code
CARD-EMBOSSED-NAME    PIC X(50)  -- Name printed on card
CARD-EXPIRAION-DATE   PIC X(10)  -- YYYY-MM-DD
CARD-ACTIVE-STATUS    PIC X(01)  -- 'Y' active / 'N' inactive

✅ Acceptance Criteria Examples

  • Given authenticated user with account in COMMAREA → CCLI displays all cards for that account
  • Given card number selected on list → CCDL shows full card details (number masked except last 4)
  • Given valid update to CARD-EMBOSSED-NAME → CARDFILE updated, confirmation displayed
  • Given card deactivation (CARD-ACTIVE-STATUS = 'N') → Card cannot be used for new transactions (validated by POSTTRAN)
  • Given invalid card number format → Error message displayed, no CARDFILE write
