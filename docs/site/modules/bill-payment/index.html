💰 Bill Payment Module

CardDemo — User Story Development Guide

📋 Overview

Purpose: Allows cardholders to pay their credit card balance online through a CICS transaction that immediately updates account balances and creates an audit transaction record.

Business Context: Bill payment is a real-time online credit to the cardholder's account. It reduces the outstanding balance and creates an audit trail in TRANFILE. Payment applies to the current balance.

CICS Transaction: CB00 | BMS Map: COBIL00 | Program: COBIL00C

🎯 User Story Development Guide

📝 US Templates

  • As a cardholder, I want to pay my credit card bill online so that I can reduce my outstanding balance immediately
  • As a cardholder, I want to see my current balance before making a payment so that I know how much I owe
  • As a cardholder, I want to receive a payment confirmation so that I know the payment was processed
  • As a financial auditor, I want bill payment transactions in TRANFILE so that all credits are traceable
  • As a cardholder, I want ACCT-CURR-CYC-CREDIT updated on payment so that the cycle credit total is accurate

⚡ Development Acceleration Factors

  • Account update pattern: ACCTFILE read-modify-write established in COACTUPC
  • Transaction write pattern: TRANFILE write established in COTRN02C
  • BMS map COBIL00: Payment screen layout pre-defined

📊 Complexity Guidelines

  • 1–2 pts Display payment screen with current balance from ACCTFILE
  • 3–5 pts Payment processing: validate, update ACCTFILE, write TRANFILE, confirmation
  • 5–8 pts Partial payment support, payment scheduling, EFT account integration

🔧 Technical Foundation

🏗️ Key Components

ComponentTypeResponsibility
COBIL00CCICS ProgramBill payment: reads account, applies payment, writes transaction record
ACCTFILEVSAM KSDSAccount master — ACCT-CURR-BAL and ACCT-CURR-CYC-CREDIT updated
TRANFILEVSAM KSDSTransaction master — payment transaction record written for audit

📊 Payment Processing Logic

1. Read ACCTFILE by ACCT-ID (from COMMAREA)
2. Display current ACCT-CURR-BAL on COBIL00 screen
3. User enters payment amount (or system defaults to full balance)
4. Validate: payment > 0, payment ≤ ACCT-CURR-BAL
5. Update ACCTFILE:
     ACCT-CURR-BAL       = ACCT-CURR-BAL - payment amount
     ACCT-CURR-CYC-CREDIT = ACCT-CURR-CYC-CREDIT + payment amount
6. Write payment transaction to TRANFILE (type = payment)
7. Display confirmation with new balance

✅ Acceptance Criteria Examples

  • Given authenticated cardholder → CB00 screen shows current account balance
  • Given valid payment amount → ACCT-CURR-BAL reduced, ACCT-CURR-CYC-CREDIT increased, confirmation shown
  • Given payment > current balance → Error message displayed, no VSAM updates performed
  • Given zero payment amount → Error message displayed, no processing
  • Given successful payment → New transaction record appears in TRANFILE with payment type code
  • Given successful payment → Updated balance visible immediately on subsequent Account View (CAVW)
