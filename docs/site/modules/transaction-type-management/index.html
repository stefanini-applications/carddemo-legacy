📦 Transaction Type Management

User story guide for the DB2-backed transaction metadata module.

📋 Overview

Purpose: Enable admins to CRUD transaction-type reference data in DB2 and keep VSAM caches (used by posting jobs) synchronized.

Business Context: Transaction-type metadata drives routing and rewards logic; this module shows how to manage that metadata relationally while feeding the high-performance VSAM posting engine.

🎯 User Story Development Guide

📝 Module-Specific US Templates

  • Add a type: As an admin, I want to add a new transaction type via `CTTU` so the posting job can accept the new code.
  • Batch export: As an operator, I want `TRANEXTR` to emit fresh VSAM files after DB2 updates so that the next posting run uses current data.

⚡ Development Acceleration Factors

  • DB2 DDL templates: `app/app-transaction-type-db2/ddl` already defines `CARDDEMO.TRANSACTION_TYPE` and `TRANSACTION_TYPE_CATEGORY`—reuses those for new stories.
  • Batch extraction: `TRANEXTR`/`CREADB21` jobs provide automation for replicating DB2 tables to VSAM, reducing manual interventions.

📊 Complexity Guidelines

  • Simple: Update the `CTTU` map to include an informational field.
  • Medium: Add validation logic that ensures a transaction type description is not blank before hitting DB2.
  • Complex: Change DB2 schema (new column or constraint) and re-run `CREADB21`, then refresh `TRANEXTR` to ship the data to VSAM.

🔧 Technical Foundation

🏗️ Key Components

  • CICS Programs: `COTRTUPC` (add/edit via `CTTU`), `COTRTLIC` (list/update/delete via `CTLI`).
  • DB2 Artifacts: `CARDDEMO.TRANSACTION_TYPE`, `CARDDEMO.TRANSACTION_TYPE_CATEGORY`, `dsntiapl` precompiler output, SQLCA error blocks in `app/app-transaction-type-db2/dcl`.
  • Batch Jobs: `CREADB21` (creates table structure), `TRANEXTR` (exports DB2 data to VSAM), `MNTTRDB2` (batch maintenance program `COBTUPDT`).

🔗 Dependencies

  • DB2 subsystem with precompiler; `CREADB21` uses DSNTEP4, `TRANEXTR` uses DSNTIAUL.
  • VSAM posting files refreshed by `TRANEXTR` so `CT00`/`CT01` never work against stale data.
  • Base CardDemo menu (`CA00`) exposes options 5 and 6 only after this module is installed.

🌐 Public Interfaces


CICS Transactions:
  CTTU - Transaction Type add/edit
  CTLI - Transaction Type list/update/delete
Batch Jobs:
  CREADB21 - Create DB2 tables and load data
  TRANEXTR - Extract DB2 tables into VSAM format
  MNTTRDB2 (COBTUPDT) - Batch maintenance

✅ Acceptance Criteria Examples

  • New entries submitted via `CTTU` reach DB2 (`CARDDEMO.TRANSACTION_TYPE`) and appear after running `TRANEXTR` in the VSAM posting dataset.
  • Deletes triggered in `CTLI` respect the `DELETE RESTRICT` rule in `CARDDEMO.TRANSACTION_TYPE_CATEGORY` and emit a meaningful error message.
  • `TRANEXTR` runs nightly without locking VSAM files and leaves `TRANTYPE` dataset ready for the next `POSTTRAN` batch.
