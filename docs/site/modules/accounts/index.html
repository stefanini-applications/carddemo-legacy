🏦 Accounts Module

CardDemo — User Story Development Guide

📋 Overview

Purpose: Provides cardholders with the ability to view and update their credit card account information, including balances, limits, and account status.

Business Context: The account is the central entity in CardDemo — it links customers, cards, and transactions. Account balances and credit limits govern transaction approval and interest calculation.

CICS Transactions: CAVW (view), CAUP (update)

Batch Jobs: INTCALC (CBACT04C), ACCTFILE, CBACT01C–CBACT03C

🎯 User Story Development Guide

📝 US Templates

  • As a cardholder, I want to view my account balance and credit limit so that I can manage my spending
  • As a cardholder, I want to view my account open and expiration dates so that I know when my card will be reissued
  • As a cardholder, I want to update my billing ZIP code so that my address information is current
  • As a batch operator, I want to run INTCALC so that interest is applied to all active account balances each cycle
  • As a financial analyst, I want current-cycle credit and debit totals so that I can monitor spending patterns

⚡ Development Acceleration Factors

  • CVACT01Y copybook: Complete account record structure pre-defined
  • COACTVWC pattern: Read-only view pattern with VSAM KSDS access established
  • DISCGRP linkage: ACCT-GROUP-ID → DISCGRP rate lookup pattern available in CBACT04C

📊 Complexity Guidelines

  • 1–2 pts Account View (CAVW) — display existing record from ACCTFILE
  • 3–5 pts Account Update (CAUP) — validate and write back to ACCTFILE
  • 5–8 pts Interest calculation batch integration; account group/rate rule changes

📊 Account Data Model

ACCT-ID                  PIC 9(11)      -- Primary key
ACCT-ACTIVE-STATUS       PIC X(01)      -- 'Y' active / 'N' inactive
ACCT-CURR-BAL            PIC S9(10)V99  -- Current balance
ACCT-CREDIT-LIMIT        PIC S9(10)V99  -- Total credit line
ACCT-CASH-CREDIT-LIMIT   PIC S9(10)V99  -- Cash advance sub-limit
ACCT-OPEN-DATE           PIC X(10)      -- YYYY-MM-DD
ACCT-EXPIRAION-DATE      PIC X(10)      -- YYYY-MM-DD
ACCT-REISSUE-DATE        PIC X(10)      -- YYYY-MM-DD
ACCT-CURR-CYC-CREDIT     PIC S9(10)V99  -- Cycle credits (payments)
ACCT-CURR-CYC-DEBIT      PIC S9(10)V99  -- Cycle debits (purchases)
ACCT-ADDR-ZIP            PIC X(10)      -- Billing ZIP
ACCT-GROUP-ID            PIC X(10)      -- Links to DISCGRP rates

✅ Acceptance Criteria Examples

  • Given authenticated user with valid account ID in COMMAREA → CAVW displays all account fields within 2s
  • Given account update with valid ZIP code → ACCTFILE record updated, confirmation message displayed
  • Given account update with invalid field → Error message displayed, no VSAM write performed
  • Given inactive account (ACCT-ACTIVE-STATUS = 'N') → View shows status, update may be restricted
  • Given INTCALC batch runs → All active accounts have interest applied based on DISCGRP rates for their group
  • Given INTCALC completes → ACCT-CURR-BAL reflects accrued interest for each account
