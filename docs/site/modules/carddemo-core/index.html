­ƒôï Overview

Purpose: Host the baseline credit-card lifecycle (sign-on, account management, transactions, billing, reporting, and the nightly batch jobs that refresh the VSAM datasets).

Business Context: CardDemo emulates a legacy mainframe bank platform; core stories keep cardholder experiences aligned with the nightly VSAM refresh and BMS-driven UI.

­ƒôØ Module-Specific US Templates

  • Account visibility: As a cardholder, I want to view my transactions (CT00/CT01) so that I can reconcile spending before the next statement.
  • Billing operations: As an officer, I want POSTTRAN + CREASTMT to run nightly so that statements portray the latest balances.
  • Security controls: As an admin, I want CU02/CU03 workflows to only succeed under an active CC00 session so that user changes remain auditable.

ÔÜí Development Acceleration Factors

  • Shared copybooks: `app/cpy/`, `app/cpy-bms/` hold structures like `CVACT01Y`, reducing repeated layout work.
  • Base JCL templates: Jobs such as `ACCTFILE`, `TRANIDX`, and `CREASTMT` in `app/jcl/` can be copied/adapted for new nightly tasks.
  • BMS map skeletons: Maps under `app/bms/` (e.g., `COACTUP`, `COMEN01`) provide ready-made regions for new navigation or messaging zones.

­ƒôè Story Complexity Guidelines

  • Simple (1-2 pts): Add a validation message to an existing BMS map (e.g., error text on CAUP).
  • Medium (3-5 pts): Create a new transaction that reads a slightly different VSAM file and returns it to a new map.
  • Complex (5-8 pts): Introduce a new account lifecycle workflow that spans CAUP ÔåÆ POSTTRAN ÔåÆ TRANIDX, ensuring nightly synchronization.

Ô£à Acceptance Criteria Examples

  • CAUP saves only when each field matches the copybook length and `IF ACCOUNT-BALANCE >= 0` passes.
  • Transaction list CT00 returns rows paged via `CTLI` and never leaves duplicate entries when TRANEXTR refresh runs simultaneously.
  • Batch job CREASTMT creates statements and signals TRANIDX before OPENFIL unlocks VSAM files.

­ƒöº Technical Foundation

­ƒÅù´©Å Key Components

  • COBOL programs: `COSGN00C`, `COACTUP`, `COTRN00C`, `COPAUS0C` orchestrate sign-on, account updates, transactions, and pending authorizations.
  • BMS maps: Mapsets such as `COACTVW`, `COMEN01`, `COPAU00` render the terminal UI and embed validation prompts.
  • VSAM datasets: `ACCTDATA`, `CARDDATA`, `TRANSACT`, `DALYTRAN` hold production data; copybooks in `app/cpy/` define their pictures.
  • Batch jobs: `POSTTRAN`, `TRANEXTR`, `CREASTMT`, `TRANIDX`, `OPENFIL` orchestrate the overnight refresh.

­ƒöù Dependencies

  • CICS resources defined in `app/csd/` (program/table definitions, transaction IDs, tasks).
  • Copybook library (`app/cpy/`, `app/cpy-bms/`) reused across screens for shared fields.
  • Scheduler utilities (`app/scheduler/`) and dataset loaders under `app/proc/`/`app/maclib/`.
