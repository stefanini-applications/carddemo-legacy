📊 Reports

Transaction Reports & Statement Generation via Internal Reader

📋 Overview

Module ID: reports

Purpose: Enables online submission of transaction report requests (CR00). Reports are generated as batch jobs submitted via the CICS internal reader (JES). Monthly statements produced by CBSTM03A; detailed transaction reports by CBTRN03C.

Business Context: Reporting is essential for customer billing, reconciliation, and audit. CORPT00C collects date range from user and submits JCL to the internal reader, which runs the report batch job asynchronously.

🎯 User Story Templates

As a regular user, I want to request a transaction report for a date range so that I can review my activity.
As a regular user, I want the system to default to yesterday's date so that I don't have to enter dates manually for a daily report.
As a batch system, I want to produce monthly statements (CBSTM03A) so that customers receive billing documents.
As a batch system, I want to generate a detailed transaction report (CBTRN03C) so that operations has a complete transaction log.

📊 Story Complexity

Simple 1-2 pts Medium 3-5 pts Complex 5-8 pts

Simple (1-2 pts)

  • Display report request screen (CR00)
  • Default start date = current date - 1

Medium (3-5 pts)

  • Validate date range (start ≤ end)
  • Submit JCL to internal reader via EXEC CICS SPOOLWRITE
  • Display submission confirmation

Complex (5-8 pts)

  • CREASTMT (CBSTM03A/B): full monthly statement generation
  • Report output to PDF via TXT2PDF JCL utility
  • Email/distribute statements to customers

✅ Acceptance Criteria Patterns

Screen Load (CR00): When report screen loads, start date defaults to yesterday (FUNCTION DATE-OF-INTEGER) and end date defaults to today.
Date Validation: Given start date after end date, then error message is displayed and job is NOT submitted.
Job Submission: Given valid date range and user presses ENTER, then JCL is submitted to internal reader via EXEC CICS, and confirmation message with job name is displayed.
CBSTM03A Output: Given completed batch run, then statement file is produced for each account with period start/end dates, all transactions, beginning/ending balance.
PF3: Returns to main menu (CM00) without submitting any job.

Date Calculation Pattern (CORPT00C)

COMPUTE WS-CURDATE-N = 
    FUNCTION DATE-OF-INTEGER(
        FUNCTION INTEGER-OF-DATE(WS-CURDATE-N) - 1)
-- Calculates yesterday's date using intrinsic functions

🔗 Dependencies

  • transactions: TRANFILE is the primary input for CBTRN03C
  • accounts: ACCTFILE used by CBSTM03A for balance info
  • batch-processing: CREASTMT is part of the end-of-day pipeline
  • JES internal reader: Required for online job submission from CICS

⚡ Development Acceleration

  • FUNCTION DATE-OF-INTEGER: Date arithmetic pattern established — reuse for any date calculation
  • EXEC CICS SPOOLWRITE pattern: Internal reader submission established in CORPT00C
  • CBSTM03B: Subprogram pattern for statement formatting — extend for new report formats
