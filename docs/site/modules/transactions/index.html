📋 Transactions

User Story Development Guide — CardDemo Mainframe Application

Overview

Supports online transaction listing (CT00), detail view (CT01), and manual add (CT02). Transactions are stored in the TRANSACT VSAM KSDS (350-byte records) with an Alternate Index on card number. Daily transactions arrive via the DALYTRAN sequential file and are batch-posted by CBTRN02C. Transaction type and category codes are validated against reference data files.

Writing Good Stories for This Module

  • Reference TRAN-TYPE-CD and TRAN-CAT-CD validation in add/update stories.
  • Manual add stories should specify that type and category codes must exist in TRANTYPE and TRANCATG.
  • Signed amount behavior must be specified: positive = debit (charge), negative = credit (refund/payment).
  • Search-by-card-number stories use the AIX on TRANSACT — note the browse vs. random access distinction.

Complexity Grid

🟢 Simple (1–2 pts)

  • List/view transactions (read-only, existing screens)
  • Paginate transaction list (scroll forward/back)
  • Display transaction amount sign convention on screen

🟡 Medium (3–5 pts)

  • Manually add transaction with type/category validation
  • Filter transaction list by type code
  • Sort transaction list by date or amount

🔴 Complex (5–8 pts)

  • Transaction search by date range (AIX browse with range logic)
  • Dispute workflow: flag transaction + update status + notify
  • Batch posting reconciliation report (DALYTRAN vs. TRANSACT comparison)

Business Rules

  • Amount sign: TRAN-AMT is signed — positive values represent debits (charges); negative values represent credits (payments, refunds).
  • Rejected transactions: Daily transactions that fail validation during batch posting are written to DALYREJS sequential file with a reason code.
  • Reference validation: TRAN-TYPE-CD and TRAN-CAT-CD must exist in TRANTYPE and TRANCATG reference files respectively.
  • Card number AIX: TRANSACT has an Alternate Index on TRAN-CARD-NUM enabling browsing transactions by card without full file scan.
  • Batch posting: CBTRN02C processes DALYTRAN and updates both TRANSACT and ACCTFILE balances.
