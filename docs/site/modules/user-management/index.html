👥 User Management

Admin-Only CRUD for Application Users — CU00-CU03 / USRSEC VSAM

📋 Overview

Module ID: user-management

Purpose: Admin-only module for managing CardDemo application users. Provides full CRUD: list all users (CU00), add user (CU01), update user (CU02), delete user (CU03). All operations affect the USRSEC VSAM security file.

Business Context: Controls who can access the CardDemo application and at what privilege level. Admin users manage the user population. Access is restricted: non-admin users attempting to access admin menu are redirected.

⚠️ Admin Only: All screens check CDEMO-USER-TYPE = 'A' in COMMAREA. Regular users are redirected to main menu.

🎯 User Story Templates

As an admin, I want to list all application users so that I have visibility into who has access.
As an admin, I want to add a new user with a specific type (admin/regular) so that new staff can access the system.
As an admin, I want to update a user's password or type so that I can manage access privileges.
As an admin, I want to delete a user so that terminated employees lose application access.
As the system, I want to reject non-admin users from the admin menu so that regular users cannot manage other users.

✅ Acceptance Criteria Patterns

Admin Access Check: Given CDEMO-USER-TYPE ≠ 'A' in COMMAREA, when any admin screen is accessed, then user is redirected to main menu (CM00) with access denied message.
User List (CU00): When CU00 loads, then all users in USRSEC are listed showing: user ID, first name, last name, user type. Paginate with PF7/PF8.
Add User (CU01): Given unique user ID, names, password, and valid type ('A' or 'U'), when ENTER pressed, then record written to USRSEC and success message shown.
Duplicate User ID: Given user ID that already exists in USRSEC, then DUPLICATE record error is displayed and no write is performed.
Update User (CU02): Given admin modifies user type or password, when ENTER pressed, then USRSEC record is rewritten and success message shown.
Delete User (CU03): Given valid user ID, when confirmed and ENTER pressed, then record is deleted from USRSEC via EXEC CICS FILE DELETE.
