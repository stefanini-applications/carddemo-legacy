📋 Module Overview

Purpose: Controls user signon/signoff and session initialization for the CardDemo application.

Business Context: All access to CardDemo begins at the signon screen. Authentication determines user type (Regular vs Admin) and routes users to the appropriate menu. Without successful authentication, no other function is accessible.

Transaction: CC00  |  Program: COSGN00C  |  BMS Map: COSGN00

COBOL CICS VSAM USRSEC BMS COMMAREA

🎯 User Story Templates

US-AUTH-001: As a cardholder, I want to log in with my User ID and Password so that I can access my account functions.
US-AUTH-002: As an administrator, I want to log in so that I can access the admin menu and manage users.
US-AUTH-003: As a cardholder, I want to see an error message when my credentials are invalid so that I know my login failed.
US-AUTH-004: As a system operator, I want the signon screen to clear the password field after a failed attempt so that credentials are not exposed.

✅ Acceptance Criteria Patterns

Valid login (Regular User):
GIVEN a valid regular user ID and password
WHEN the user submits the signon screen
THEN the Main Menu (CM00) is displayed
Valid login (Admin User):
GIVEN a valid admin user ID and password
WHEN the user submits the signon screen
THEN the Admin Menu (CA00) is displayed
Invalid credentials:
GIVEN an invalid user ID or password
WHEN the user submits the signon screen
THEN an error message is displayed and the user remains on the signon screen

Data Model – SEC-USER-DATA (CSUSR01Y)

01 SEC-USER-DATA.
   05 SEC-USR-ID       PIC X(08)   -- User ID (8 chars, space-padded)
   05 SEC-USR-FNAME    PIC X(20)   -- First name
   05 SEC-USR-LNAME    PIC X(20)   -- Last name
   05 SEC-USR-PWD      PIC X(08)   -- Password (8 chars)
   05 SEC-USR-TYPE     PIC X(01)   -- R = Regular, A = Admin
   05 SEC-USR-FILLER   PIC X(23)

Authentication Flow

  1. User starts CC00 transaction in CICS terminal
  2. COSGN00C displays BMS map COSGN00
  3. User enters User ID and Password, presses Enter
  4. Program reads USRSEC VSAM file keyed by User ID
  5. Compares entered password with SEC-USR-PWD
  6. On match: checks SEC-USR-TYPE
  7. Type R → XCTL to COMEN01C (CM00 Main Menu)
  8. Type A → XCTL to COADM01C (CA00 Admin Menu)
  9. On failure: display error message on signon screen

Dependencies

  • VSAM USRSEC: Must be loaded via DUSRSECJ batch job before online use
  • CICS Resources: Transaction CC00 and mapset COSGN00 must be defined in CICS CSD
  • COMMAREA: Session state flows through CICS COMMAREA to all subsequent programs

⚡ Development Acceleration Factors

  • CSUSR01Y copybook: Standard user record layout, reuse for all user security operations
  • COCOM01Y copybook: Common commarea pattern shared across all CICS programs
  • DFHAID / DFHBMSCA: Standard CICS key handling copybooks included in all programs
  • CC00 is the entry point: All new online user stories begin with an authenticated session from this module
