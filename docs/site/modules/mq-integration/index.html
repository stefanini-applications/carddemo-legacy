📋 Module Overview

Purpose: Demonstrates asynchronous account data extraction and system date inquiry through IBM MQ request/response patterns. Shows how CICS programs can communicate with external systems via MQ queues.

Business Context: Modern mainframe environments frequently integrate with distributed systems via MQ. This module demonstrates the request/response MQ pattern where a CICS program sends a request message and waits for a response - applicable to account inquiries, date services, or any external data lookup.

Transactions: CDRD (system date inquiry) | CDRA (account details inquiry)
Programs: CODATE01 / COACCT01

COBOL CICS IBM MQ VSAM Request/Response Pattern Optional Module

🎯 User Story Templates

US-MQ-001: As an integration developer, I want to query the system date via MQ so that external systems receive the mainframe system time in a standard format.
US-MQ-002: As an integration developer, I want to query account details via MQ so that external systems can retrieve account information asynchronously.
US-MQ-003: As an architect, I want to demonstrate the MQ request/response pattern so that the team understands how to integrate CICS with external message-based systems.
US-MQ-004: As a developer, I want account data returned via MQ to include balance and limit information so that external systems have complete account context.

✅ Acceptance Criteria Patterns

Date Inquiry (CDRD):
GIVEN the CDRD transaction is invoked
WHEN CODATE01 sends the MQ request
THEN a response message containing the current system date is received from the reply queue
Account Inquiry (CDRA):
GIVEN a valid account ID is provided in the MQ request
WHEN COACCT01 processes the request
THEN account details are read from VSAM ACCTFILE and returned via MQ response message
MQ Timeout Handling:
GIVEN no response arrives within the configured timeout
WHEN the wait period expires
THEN an appropriate error/timeout message is returned to the caller

MQ Request/Response Pattern

  1. External client (or CICS transaction) puts message on MQ request queue
  2. CICS transaction CDRD or CDRA is triggered (or initiated directly)
  3. Program reads the request message from the MQ queue
  4. For CODATE01: retrieves system date and formats response
  5. For COACCT01: reads ACCTFILE VSAM with account ID from request
  6. Program puts response message on MQ reply queue
  7. External client receives the response from the reply queue

MQ Configuration Requirements

  • IBM MQ configured and accessible from CICS region
  • CICS with MQ support (CKTI or MQ-CICS Bridge)
  • Request and reply MQ queues defined
  • CICS resources defined for CDRD and CDRA transactions
  • CSD definitions in app/app-vsam-mq/csd/

Dependencies

  • IBM MQ Infrastructure: MQ must be running and queues defined
  • accounts module: COACCT01 reads ACCTFILE for account data
  • authentication: CICS session may be required depending on transaction security settings

⚡ Development Acceleration Factors

  • CODATE01 MQ pattern: Simplest MQ request/response - use as starting template for new MQ-integrated programs
  • COACCT01 MQ+VSAM pattern: Demonstrates combining MQ messaging with VSAM data access
  • CSD template: CICS resource definitions in csd/ directory provide reusable MQ transaction setup
