💳 Credit Cards

User Story Development Guide · CardDemo Mainframe Application

Core Module CCLI · CCDL · CCUP

📋 Overview

The Credit Cards module provides online listing, viewing, and updating of credit cards associated with an account. Cards are the primary instrument for transaction authorization and are linked to accounts via the CARDXREF cross-reference VSAM.

CICS Transactions

Transaction IDProgramFunction
CCLICOCRDLICList all cards for an account (paginated)
CCDLCOCRDSLCView detailed card information
CCUPCOCRDUPCUpdate card fields (embossed name, status)

Data Stores

CARDDATA VSAM KSDS — 150-byte card records keyed on CARD-NUM (16 chars), defined in copybook CVACT02Y.

CARDXREF VSAM KSDS — cross-reference records linking card numbers to account IDs, defined in copybook CVACT03Y. An Alternate Index (AIX) on CARDXREF enables lookup by account ID, supporting the card-list-by-account navigation pattern.

Business Context

Multiple cards may be associated with a single account. The card's CARD-ACTIVE-STATUS field (A=Active, I=Inactive) is checked by COTRN02C before any transaction is accepted — inactive cards are declined at the point of entry.

The AIX on CARDXREF enables efficient navigation from an account ID to all associated card numbers without a full file scan, which underpins the CCLI list transaction.

🎯 User Story Development Guide

Story Templates

List All Cards for Account
As a cardholder, I want to see all credit cards linked to my account so that I can manage each card individually.

Program: COCRDLIC (CCLI) — reads CARDXREF AIX by account ID, displays paginated list of CARD-NUM + CARD-ACTIVE-STATUS
View Card Details
As a cardholder, I want to view the full details of a specific card including expiry date and embossed name so that I can confirm card information.

Program: COCRDSLC (CCDL) — keyed read on CARDDATA VSAM using CARD-NUM; CVV not displayed on screen
Activate / Deactivate Card
As a customer service representative, I want to change a card's active status so that lost or replaced cards can be deactivated immediately.

Program: COCRDUPC (CCUP) — read-for-update → set CARD-ACTIVE-STATUS → rewrite to CARDDATA VSAM
Update Embossed Name
As a customer service representative, I want to update the embossed name on a card so that name-change requests are reflected in the system.

Field: CARD-EMBOSSED-NAME PIC X(50) — updated via COCRDUPC (CCUP)

Acceleration Factors

  • CVACT02Y copybook — fully defined 150-byte card record; no data modeling required
  • CVACT03Y xref lookup pattern — AIX-based account-to-card navigation already implemented in COCRDLIC
  • BMS maps exist — COCRDLI.bms, COCRDSLI.bms, COCRDUPD.bms ready for modification
  • AIX on CARDXREF — enables account-to-card navigation without file scan; pattern reusable for any account-scoped list

✅ Acceptance Criteria Patterns

Card List (CCLI)

  • GIVEN an authenticated user with a valid account ID, WHEN CCLI is invoked, THEN all cards associated with that account are listed via CARDXREF AIX lookup
  • GIVEN an account with more than one page of cards, WHEN the user pages forward, THEN the next set of cards is displayed correctly
  • GIVEN an account with no associated cards, WHEN CCLI is invoked, THEN an appropriate "no cards found" message is displayed

Card Detail (CCDL)

  • GIVEN a valid card number, WHEN CCDL is invoked, THEN all card details are displayed accurately from CARDDATA VSAM
  • GIVEN a card detail screen, WHEN it is displayed, THEN CVV (CARD-CVV-CD) is not shown or is masked

Card Update (CCUP)

  • GIVEN a valid status change request, WHEN CCUP processes it, THEN CARD-ACTIVE-STATUS is updated in CARDDATA VSAM and a confirmation is displayed
  • GIVEN a name update, WHEN CCUP processes it, THEN CARD-EMBOSSED-NAME is updated and the change is immediately visible on CCDL
  • GIVEN an invalid status value, WHEN CCUP validates input, THEN the update is rejected with a clear validation error
