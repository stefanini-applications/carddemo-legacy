🏦 Accounts

Account View, Update & Batch Interest Calculation

📋 Overview

Module ID: accounts

Purpose: Provides online view and update of credit card account records stored in ACCTFILE VSAM. Includes batch programs for account maintenance (CBACT01C-03C) and interest calculation (CBACT04C) using disclosure group rates.

Business Context: Central module linking customers, cards, and transactions. Account balance, credit limit, and group ID are key data points used across bill payment, transactions, and interest calculations.

🎯 User Story Templates

As a regular user, I want to view my account balance and credit limit so that I know how much credit is available.
As a regular user, I want to update my account ZIP code so that my address is accurate.
As a batch system, I want to calculate monthly interest charges using disclosure group rates so that balances reflect accrued interest.
As a regular user, I want to see both purchase and cash credit limits so that I understand my credit structure.

📊 Story Complexity

Simple 1-2 pts Medium 3-5 pts Complex 5-8 pts

Simple (1-2 pts)

  • Display account view screen (CAVW/COACTVWC)
  • Read account record from ACCTFILE by ACCT-ID

Medium (3-5 pts)

  • Update account fields and write back to ACCTFILE (CAUP)
  • Validate credit limit changes against current balance
  • CBACT01C-03C: account file maintenance batch

Complex (5-8 pts)

  • INTCALC (CBACT04C): full interest calc pipeline across TCATBALF + DISCGRP
  • Credit limit increase workflow with approval rules
  • Account reissue / expiration processing

✅ Acceptance Criteria Patterns

Account View (CAVW): Given valid ACCT-ID in COMMAREA, when screen loads, then all account fields are displayed: balance, credit limit, cash limit, open date, expiry date, group ID.
Account Update (CAUP): Given user modifies allowed fields and presses ENTER, then ACCTFILE is updated via EXEC CICS FILE REWRITE and success message is displayed.
Account Not Found: Given ACCT-ID not in ACCTFILE, then error message is displayed and user is returned to previous screen.
Interest Calc (INTCALC): Given TCATBALF has balances for account/type/category, then CBACT04C reads DISCGRP rate and applies: new balance = old balance × (1 + rate/12).
Active Status: ACCT-ACTIVE-STATUS must be 'Y' for account to be considered active; inactive accounts display appropriate status indicator.

Account Record Structure (CVACT01Y.cpy)

01  ACCOUNT-RECORD.                       (RECLN 300)
    05  ACCT-ID                PIC 9(11)   -- primary key
    05  ACCT-ACTIVE-STATUS     PIC X(01)   -- 'Y'=active
    05  ACCT-CURR-BAL          PIC S9(10)V99
    05  ACCT-CREDIT-LIMIT      PIC S9(10)V99
    05  ACCT-CASH-CREDIT-LIMIT PIC S9(10)V99
    05  ACCT-OPEN-DATE         PIC X(10)   -- YYYY-MM-DD
    05  ACCT-EXPIRAION-DATE    PIC X(10)   -- note: typo preserved
    05  ACCT-REISSUE-DATE      PIC X(10)
    05  ACCT-CURR-CYC-CREDIT   PIC S9(10)V99
    05  ACCT-CURR-CYC-DEBIT    PIC S9(10)V99
    05  ACCT-ADDR-ZIP          PIC X(10)
    05  ACCT-GROUP-ID          PIC X(10)   -- links to DISCGRP

🔗 Dependencies

  • authentication: COMMAREA must contain CDEMO-ACCT-ID
  • XREFFILE: Card-to-account cross-reference (used by transactions)
  • DISCGRP VSAM: Required by CBACT04C for interest rate lookup
  • TCATBALF VSAM: Transaction category balances for interest calc
  • bill-payment: Updates ACCT-CURR-BAL via REWRITE
  • transactions: Reads account via XREF card number lookup

⚡ Development Acceleration

  • CVACT01Y.cpy: Reuse account structure across all programs that touch ACCTFILE
  • EXEC CICS FILE REWRITE: Standard update pattern in COACTUPC.cbl
  • EXEC CICS FILE READ UPDATE: Required before REWRITE — established pattern
  • Batch sequential read pattern: CBACT04C iterates TCATBALF — reusable for any sequential batch process
