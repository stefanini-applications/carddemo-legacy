📦 vsam-mq-extraction

Guide for MQ-driven extraction stories that surface VSAM data.

📋 Overview

Purpose: Demonstrate asynchronous MQ links to VSAM data for system date and account inquiries.

Business Context: This module serves as a template for partner systems to request time/account details without direct VSAM access.

🎯 User Story Development Guide

📝 Module-Specific US Templates

  • System Date: As a scheduler, I want to query the system date via MQ (`CDRD`) so downstream systems can align runtimes.
  • Account Lookup: As an external service, I want to send an MQ request with an account number (`CDRA`) and receive the VSAM-backed payload.
  • MQ Correlation: As an operator, I want to confirm that replies reuse the same REQUEST-ID so batch consumers can correlate replies.

⚡ Development Acceleration Factors

  • MQ patterns: Message definitions for date/account request/response exist in the extension README, showing the required fields and lengths.
  • COBOL programs: CODATE01 and COACCT01 already parse MQ requests and emit responses.
  • Queue definitions: `CARDDEMO.REQUEST.QUEUE` and `.RESPONSE.QUEUE` are named consistently for reuse.

📊 Complexity Guidelines

  • Simple (1-2 pts): Add a new output field (e.g., city) to the existing account response payload.
  • Medium (3-5 pts): Introduce a new MQ listener that triggers a CICS map update when a queued account update arrives.
  • Complex (5-8 pts): Extend the MQ flow to integrate with optional modules (transaction-type or authorization) and manage multiple queue managers.

✅ Acceptance Criteria Examples

  • MQ replies must include the same REQUEST-ID as the request and keep the payload within the defined record length (e.g., 300 characters for account data).
  • Account extract responses only include records that exist in VSAM ACCTDATA.
  • System date requests respond directly from CODATE01 without touching VSAM.

🔧 Technical Foundation

🏗️ Key Components

  • CICS programs CODATE01 (system date) and COACCT01 (account details).
  • MQ channel definitions under app/app-vsam-mq/ for CARDDEMO.REQUEST.QUEUE and .RESPONSE.QUEUE.
  • VSAM datasets for account & card data.

🔗 Dependencies

  • IBM MQ must be available and reachable from the CICS region.
  • The VSAM files must be populated via `core-batch` so account values exist.
  • RACF security entries must permit MQ-initiated CICS transactions.

🌐 Public Interfaces

CDRD (System Date MQ request), CDRA (Account details MQ request), CARDDEMO.REQUEST.QUEUE, CARDDEMO.RESPONSE.QUEUE, MESSAGE STRUCTURES (DATE-REQUEST/RESPONSE, ACCT-REQUEST/RESPONSE)
