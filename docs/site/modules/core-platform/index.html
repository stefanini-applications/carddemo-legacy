📦 Core Platform

Guide for developing user stories anchored on the base CardDemo experience (authentication, VSAM CRUD, bills, and batch orchestration).

📋 Overview

Purpose: Deliver the standard cardholder and admin journeys—sign-on, account/card maintenance, transaction posting, bill payments, and batch reporting—using CICS/VSAM primitives.

Business Context: This is the foundational module that every optional extension builds on, so successful user stories here unlock reliable online operations and nightly batch runs.

🎯 User Story Development Guide

📝 Module-Specific US Templates

  • Account Maintenance: As a cardholder, I want to update my billing address via the `CAUP` screen so subsequent statements reflect the new data.
  • Transaction Posting: As an operations analyst, I want `CT02` to post a debit transaction with accurate posting date so the ledger balances before nightly processing.
  • Billing & Reports: As a finance user, I want `CB00` to confirm payment success and queue the record for `CREASTMT`.

⚡ Development Acceleration Factors

  • BMS map reuse: `COSGN00`, `COMEN01`, `COACTVW`, `COTRN02` already define layout/keys for standard journeys; extend them instead of rebuilding from scratch.
  • Shared copybooks: Copybooks in `app/cpy` describe accounts, cards, and transactions; adding fields here propagates to multiple COBOL and batch programs.
  • Batch templates: JCL templates and scheduler definitions (CA7/Control-M) exist for `POSTTRAN`, `TRANEXTR`, and `CREASTMT`, making job sequencing predictable.

📊 Complexity Guidelines

  • Simple (1-2 pts): Add a new field to an existing BMS map and COBOL structure (e.g., a status flag on `CAUP`).
  • Medium (3-5 pts): Enhance `CT02` to support a new posting type that requires an extra VSAM lookup.
  • Complex (6-8 pts): Introduce a multi-step batch job change that alters `POSTTRAN` and `CREASTMT` dependencies or migrates data between VSAM files.

🔧 Technical Foundation

🏗️ Key Components

  • COBOL programs: `COSGN00C`, `COACTVWC`, `COTRN02C`, `COPAUS0C`.
  • BMS mapsets: `COSGN00`, `COMEN01`, `COACTUP`, `COTRN*`, `COPAU00`.
  • Batch jobs: `POSTTRAN`, `CREASTMT`, `TRANBKP`, `INTCALC`, `TRANIDX`.

🔗 Dependencies

  • VSAM datasets: `CARDDEMO.ACCTDATA.PS`, `CARDDATA`, `TRANSACT.VSAM.KSDS`.
  • JCL templates: `app/jcl/*` for compile and run.
  • Authentication: RACF (`ADMIN001`, `USER0001`).

🌐 Public Interfaces


CC00 - Sign-on and dispatch
CAUP / CCUP - Account/card maintenance
CT02 - Post transaction
CB00 - Bill payment
POSTTRAN / CREASTMT - Batch posting and statement generation

✅ Acceptance Criteria Examples

  • The BMS map confirms successful updates with a message area entry and PF3 to exit.
  • `CT02` updates VSAM `TRANSACT` and makes the row visible to `CT01` lists immediately.
  • `POSTTRAN` completes within the nightly window and leaves no open files (`OPENFIL`/`CLOSEFIL` pair confirms clean-up).
