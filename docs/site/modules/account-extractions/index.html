📦 Account Extractions

User story guide for VSAM-to-MQ extraction and asynchronous inquiry transactions.

📋 Overview

Purpose: Showcase simple request/response MQ patterns that extract VSAM data (system date, account details) through transactions `CDRD` and `CDRA`.

Business Context: This module is leveraged to teach integration teams how asynchronous MQ flows can extend heavy-weight VSAM assets without disrupting interactive transactions.

🎯 User Story Development Guide

📝 Module-Specific US Templates

  • System date inquiry: As a client app, I want to send `DATE-REQUEST-MSG` via `CDRD` and consume `DATE-RESPONSE-MSG` so that my distributed scheduler stays in sync with z/OS time.
  • Account lookup: As a remote service, I want to pass an account number in `ACCT-REQUEST-MSG` through `CDRA` so that VSAM data is pushed back on `CARDDEMO.RESPONSE.QUEUE`.

⚡ Development Acceleration Factors

  • Message layouts: `DATE-REQUEST-MSG`, `DATE-RESPONSE-MSG`, `ACCT-REQUEST-MSG`, and `ACCT-RESPONSE-MSG` are defined in copybooks that can be reused for new MQ-powered inquiries.
  • Consistent MQ choreography: Reuse MQCONN/MQOPEN/MQPUT/MQGET loops from `CODATE01`/`COACCT01` when extending to new requests.

📊 Complexity Guidelines

  • Simple: Add a field to `ACCT-RESPONSE-MSG` and display it on the screen returned by `CDRA`.
  • Medium: Update `COACCT01` to call another VSAM file before packaging the response.
  • Complex: Add a new MQ request type that requires new queue definitions plus new CICS-MQ resource definitions.

🔧 Technical Foundation

🏗️ Key Components

  • Programs: `CODATE01` for system date, `COACCT01` for account details.
  • MQ resources: `CARDDEMO.REQUEST.QUEUE`, `CARDDEMO.RESPONSE.QUEUE`, MQCONN `MQ01`, MQQUEUE definitions in `app/app-vsam-mq/csd` and `jcl`.
  • VSAM lookups: Use existing `account` VSAM files (e.g., `AWS.M2.CARDDEMO.ACCTDATA.PS`) with copybook `ACCOUNT-DATA`.

🔗 Dependencies

  • IBM MQ with CICS connection definitions; queue names live in `app/app-vsam-mq/jcl`.
  • VSAM account files populated by base jobs such as `ACCTFILE` and `XREFFILE`.
  • PF keys and map logic from base BMS screens for any synchronous UI counterpart.

🌐 Public Interfaces


CICS Transactions:
  CDRD - System Date inquiry (CODATE01)
  CDRA - Account detail inquiry (COACCT01)
MQ Resources:
  CARDDEMO.REQUEST.QUEUE  - input
  CARDDEMO.RESPONSE.QUEUE - responses

✅ Acceptance Criteria Examples

  • Sending `DATE-REQUEST-MSG` via `CDRD` returns `SYSTEM-DATE` within the expected z/OS time window.
  • `CDRA` responds on `CARDDEMO.RESPONSE.QUEUE` with `ACCOUNT-DATA` that matches the VSAM `AWS.M2.CARDDEMO.ACCTDATA.PS` record.
  • MQ correlation (`REQUEST-ID`/`RESPONSE-ID`) is preserved so each client can reconcile requests.
