📦 VSAM MQ Extractions

Guide for Developing User Stories

📋 Overview

Purpose: Showcase MQ-based extraction patterns so external systems can query system date/account details while reusing existing VSAM records.

Business Context: This module proves asynchronous integration readiness for US modernization initiatives focusing on MQ-based data sharing.

🎯 User Story Development Guide

📝 Module-Specific US Templates

  • Date inquiry: As a distributed service, I want `CDRD` to return the current system date via MQ so that nightly reconciliations can align with the mainframe clock.
  • Account lookup: As a modernization tester, I want `CDRA` to look up account data and return it over MQ for API validation.
  • Correlation: As a systems engineer, I want MQ correlation IDs preserved across request-response so that retries are easy to trace.

⚡ Development Acceleration Factors

  • Message schemas: Request/response structures already documented (DATE vs ACCT) and can be extended with additional fields in a consistent order.
  • Queue definitions: `CARDDEMO.REQUEST.QUEUE`/`RESPONSE.QUEUE` and MQCONN/MQQUEUE entries live in `csd/` and can be reused for new MQ clients.
  • Extraction programs: `CDRD` and `CDRA` reuse existing BMS maps and copybooks, meaning UI updates only require small changes.

📊 Complexity Guidelines

  • Simple (1-2 pts): Update a request field in `CDRA` and adjust the CSV parser accordingly.
  • Medium (3-5 pts): Add new metadata to the account response payload and ensure MQ correlation IDs stay intact.
  • Complex (5-8 pts): Introduce a second MQ listener for new datasets while updating VSAM copybooks and BMS screens.

🔧 Technical Foundation

🏗️ Key Components

  • `CDRD`/`CODATE01`: System date MQ request/response with `DATE-REQUEST-MSG`/`DATE-RESPONSE-MSG` formats.
  • `CDRA`/`COACCT01`: Account details MQ transaction referencing `CVACT01Y`/`CVCUS01Y` for VSAM lookups.
  • `csd/` MQCONN & MQQUEUE definitions for `CARDREQ`/`CARDRES` queue names.

🔗 Dependencies

  • IBM MQ manager accessible from CICS with defined queues and channels.
  • VSAM account datasets with copybook layouts for fields returned over MQ.
  • MQ correlation ID and error-handling tables, ensuring requests/responses align.

🌐 Public Interfaces

CDRD - DATE request/response (REQUEST-TYPE 'DATE')
CDRA - ACCOUNT request/response (REQUEST-TYPE 'ACCT')
MQ Queues - CARDDEMO.REQUEST.QUEUE, CARDDEMO.RESPONSE.QUEUE
CICS MQ Resources - MQCONN(MQ01), MQQUEUE(CARDREQ/CARDRES)

📌 Acceptance Criteria Examples

  • MQ messages include the mandated `REQUEST-TYPE`, `REQUEST-ID`, and payload fields; `CDRA` treats missing fields as errors and writes diagnostics to the CICS log.
  • Responses reuse the request `REQUEST-ID` (correlation) so clients can match replies.
  • VSAM lookups handle non-existent account numbers gracefully by returning structured error text rather than blank data.
