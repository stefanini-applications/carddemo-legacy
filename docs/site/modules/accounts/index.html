📋 Module Overview

Purpose: Enables viewing and updating credit card account information including balances, credit limits, and key account dates.

Business Context: Account management is a core function for both cardholders and customer service representatives. Cardholders use the view function to monitor their financial status; CSRs use the update function to apply limit changes and account modifications.

Transactions: CAVW (view)  |  CAUP (update)  |  Programs: COACTVWC / COACTUPC

COBOL CICS VSAM ACCTFILE BMS COACTVW / COACTUP

🎯 User Story Templates

US-ACCT-001: As a cardholder, I want to view my current account balance so that I can monitor how much I have spent.
US-ACCT-002: As a cardholder, I want to view my credit limit and cash credit limit so that I know my available spending power.
US-ACCT-003: As a cardholder, I want to view my account open date and expiration date so that I can plan for card renewal.
US-ACCT-004: As a customer service representative, I want to update an account's credit limit so that I can process approved limit increase requests.
US-ACCT-005: As a customer service representative, I want to update the account group ID so that correct interest rates are applied.

✅ Acceptance Criteria Patterns

Account View:
GIVEN an authenticated cardholder with a valid account ID
WHEN the CAVW transaction is executed
THEN all account fields (balance, limits, dates, group) are displayed accurately
Account Update – Success:
GIVEN a CSR has updated the credit limit with a valid numeric value
WHEN the update is submitted
THEN the ACCTFILE VSAM record is updated and a success message is displayed
Account Update – Validation:
GIVEN a CSR enters a non-numeric value in a numeric field
WHEN the update is submitted
THEN an error message is displayed and the record is not updated

Data Model – ACCOUNT-RECORD (CVACT01Y)

01 ACCOUNT-RECORD.
   05 ACCT-ID                  PIC 9(11)       -- 11-digit account ID (key)
   05 ACCT-ACTIVE-STATUS       PIC X(01)       -- Account status (A=Active)
   05 ACCT-CURR-BAL            PIC S9(10)V99   -- Current balance
   05 ACCT-CREDIT-LIMIT        PIC S9(10)V99   -- Credit limit
   05 ACCT-CASH-CREDIT-LIMIT   PIC S9(10)V99   -- Cash advance limit
   05 ACCT-OPEN-DATE           PIC X(10)       -- Open date (YYYY-MM-DD)
   05 ACCT-EXPIRAION-DATE      PIC X(10)       -- Expiration date (YYYY-MM-DD)
   05 ACCT-REISSUE-DATE        PIC X(10)       -- Reissue date (YYYY-MM-DD)
   05 ACCT-CURR-CYC-CREDIT     PIC S9(10)V99   -- Cycle credits
   05 ACCT-CURR-CYC-DEBIT      PIC S9(10)V99   -- Cycle debits
   05 ACCT-ADDR-ZIP            PIC X(10)       -- ZIP code
   05 ACCT-GROUP-ID            PIC X(10)       -- Discount group ID
   05 FILLER                   PIC X(178)

Dependencies

  • authentication: CICS session must be established via CC00 before accessing accounts
  • ACCTFILE VSAM: Must be loaded via ACCTFILE batch job
  • bill-payment: Depends on ACCTFILE for balance updates
  • batch-processing (INTCALC): CBACT04C reads ACCTFILE for interest calculation

⚡ Development Acceleration Factors

  • CVACT01Y copybook: Standard account record layout – reuse across all account-related programs
  • COACTUPC pattern: Established update pattern with CICS REWRITE; use as template for similar update screens
  • ACCTFILE KSDS: Direct keyed access by ACCT-ID enables fast single-record retrieval
