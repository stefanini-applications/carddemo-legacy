⚙️ Batch Processing

User Story Development Guide — CardDemo Mainframe Application

Overview

The nightly batch cycle handles transaction posting (CBTRN02C / POSTTRAN), interest calculation (CBACT04C / INTCALC), statement generation (CBSTM03A / CREASTMT), report generation (CBTRN03C / TRANREPT), and file maintenance (close/open). Uses JCL, SORT, IDCAMS, and COBOL batch programs. Scheduler definitions available in app/scheduler/ (CA7 and Control-M formats).

Nightly Batch Job Sequence

1. CLOSEFIL — Close CICS-managed files for batch update
2. ACCTFILE — Account file refresh / backup
3. CARDFILE — Card file refresh / backup
4. CUSTFILE — Customer file refresh / backup
5. XREFFILE — Cross-reference file refresh / backup
6. TRANBKP — Transaction file backup (GDG)
7. TRANCATG — Transaction category refresh
8. TRANTYPE — Transaction type refresh
9. DISCGRP — Disclosure group / interest rate refresh
10. TCATBALF — Category balance file refresh
11. DUSRSECJ — User security file refresh
12. POSTTRAN (CBTRN02C) — Post daily transactions to accounts
13. INTCALC (CBACT04C) — Calculate and apply interest
14. COMBTRAN — Combine transaction files
15. CREASTMT (CBSTM03A) — Generate account statements
16. TRANREPT (CBTRN03C) — Generate transaction report
17. OPENFIL — Re-open CICS-managed files

Writing Good Stories for This Module

  • Always specify which batch job name (e.g., POSTTRAN, INTCALC) is being referenced, not just the program name.
  • Job dependency stories must reference the sequence — CLOSEFIL before file updates, OPENFIL last.
  • GDG stories should note that generation versioning requires IDCAMS define steps.
  • Stories involving DISCGRP interest rates must specify the link via ACCT-GROUP-ID.

Complexity Grid

🟢 Simple (1–2 pts)

  • Add a new SORT step to combine two sequential files
  • Run file refresh JCL (CLOSEFIL/OPENFIL cycle)
  • Add a new reason code to DALYREJS reject processing

🟡 Medium (3–5 pts)

  • Modify interest calculation rate lookup in CBACT04C
  • Add a new reject reason code with corresponding validation
  • Parameterize POSTTRAN to support multiple processing dates

🔴 Complex (5–8 pts)

  • New batch program for export to S3 (cloud modernization)
  • End-to-end batch cycle refactoring with dependency management
  • GDG version management changes (generation count, expiry)

Business Rules

  • File management: CLOSEFIL must run before any batch file refresh jobs; OPENFIL re-enables CICS file access at the end of the cycle.
  • Transaction validation: POSTTRAN validates card number via XREFFILE (CARDXREF) before posting to accounts.
  • Interest rates: INTCALC uses DISCGRP disclosure group rates; rate is selected via ACCT-GROUP-ID on the account record.
  • Rejects: Invalid DALYTRAN records are written to DALYREJS sequential file with a reason code; they are not re-processed automatically.
  • GDGs: Backup files (e.g., TRANBKP) use Generation Data Groups for versioned backups; managed via IDCAMS.
  • Scheduler: Batch scheduler definitions (CA7 and Control-M formats) are available in app/scheduler/.
