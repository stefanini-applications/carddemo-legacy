📦 VSAM MQ

Guide for building integration stories around MQ-driven account/date inquiries.

📋 Overview

Purpose: Provide MQ request/response patterns that surface VSAM account data and the system date to external clients.

Business Context: Demonstrates asynchronous integration by letting remote services query mainframe data without using terminals—critical for modernization proofs asking how MQ can replace direct screen scraping.

🎯 User Story Development Guide

📝 Module-Specific US Templates

  • Date inquiry: As a scheduler, I want to get the current mainframe date via MQ so that I can keep distributed jobs aligned.
  • Account detail: As a partner service, I want to request account information with `CDRA` and receive a 300-byte payload so I can display cardholder data in my UI.
  • Error paths: As an engineer, I want the MQ flow to surface descriptive errors if VSAM datasets are offline or MQCONN fails.

⚡ Development Acceleration Factors

  • Field layouts: `DATE-REQUEST/RESPONSE` and `ACCT-REQUEST/RESPONSE` copybooks already describe every MQ payload field.
  • CICS-MQ definitions: The `csd/` directory includes `MQCONN`, `MQQUEUE`, and `TRAN` definitions; copy them when adding new request types.
  • Reuse VSAM lookups: COBOL code references the same copybooks as the core module, so new MQ programs can call the same record parsers.

📊 Complexity Guidelines

  • Simple: Extend the `DATE-REQUEST` message with another header field and update the response filler.
  • Medium: Add a new MQ flow that returns cardholder address data by reusing VSAM account copybooks.
  • Complex: Add support for asynchronous multi-queue correlation and fallback to DB2 when VSAM is unavailable.

🔧 Technical Foundation

🏗️ Key Components

  • COBOL programs `CODATE01` and `COACCT01` under `app/app-vsam-mq/cbl` that drive MQ requests/responses.
  • Copybooks defining `DATE-REQUEST-MSG`, `DATE-RESPONSE-MSG`, `ACCT-REQUEST-MSG`, and `ACCT-RESPONSE-MSG` with correlation IDs.
  • CICS definitions (`csd/`) for MQ connections (`MQ01`) and queues (`CARDREQ`, `CARDRES`).

🔗 Dependencies

  • IBM MQ queue manager reachable from the CICS region.
  • VSAM account files (`ACCTDATA`) for lookups.
  • Core module for BMS/copybook reuse—MQ responses share field layouts with screens like `COACTVW`.

🌐 Public Interfaces

CDRD  - System date request / CODATE01 (MQ request-type 'DATE')
CDRA  - Account details request / COACCT01 (MQ request-type 'ACCT')
MQ QUEUES: CARDDEMO.REQUEST.QUEUE → CARDDEMO.RESPONSE.QUEUE
Correlation: REQUEST-ID/RESPONSE-ID fields in the payloads

✅ Acceptance Criteria Examples

  • A `CDRD` MQ request returns a response with `SYSTEM-DATE` populated and `RESPONSE-TYPE='DATE'` within 500 ms.
  • A `CDRA` request with a valid `ACCOUNT-NUMBER` returns the 300-byte account payload, and the response is correlated using the same `REQUEST-ID`.
  • If MQ is unable to connect, the COBOL program returns a descriptive error in `DFHRESP` and leaves the response queue untouched.
