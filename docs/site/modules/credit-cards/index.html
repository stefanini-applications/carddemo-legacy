🃏 Credit Cards

User Story Development Guide — CardDemo Application

📋 Overview

Purpose: List, view, and update credit card records. Each card is linked to a specific account and customer via the CARDXREF cross-reference file.

Business Context: Credit cards are the primary instrument for transactions. The card number (16 digits) links transactions, authorizations, and accounts. Card status management supports fraud prevention and lifecycle management.

COCRDLIC COCRDSLC COCRDUPC VSAM: CARDDAT VSAM: CARDXREF

🎯 User Story Development Guide

📝 Module-Specific US Templates

  • Card List: As a cardholder, I want to see all credit cards on my account so that I can identify active and inactive cards.
  • Card Detail: As a cardholder, I want to view my card details (number, status, expiration) so that I can verify card information.
  • Card Update: As an admin, I want to update a card's status so that I can block a compromised card immediately.
  • Card Navigation: As a cardholder, I want to page through a list of cards so that I can find the specific card I need.

⚡ Development Acceleration Factors

  • CARDXREF Three-Way Link: Lookup pattern CARD-NUM → CUST-ID + ACCT-ID established in COCRDSLC — reusable for all card operations.
  • Pageable List Pattern: COCRDLIC establishes forward/backward paging pattern using VSAM alternate keys — reuse for other list screens.
  • CVCRD01Y.cpy: Card work area shared across all card programs.

🔧 Technical Foundation

🏗️ Key Components

  • COCRDLIC.cbl: Card list — displays pageable list of cards for an account; selection navigates to card detail.
  • COCRDSLC.cbl: Card view/select — detailed card view using CARDXREF for cross-reference lookup.
  • COCRDUPC.cbl: Card update — modify card fields and write back to CARDDAT.
  • CVCRD01Y.cpy: Card work area copybook (holds card number, account ID, customer ID).

🔗 Dependencies

  • CARDDAT VSAM: Card master file (KSDS, key = CARD-NUM 16 digits).
  • CARDXREF VSAM: Cross-reference (KSDS, key = CARD-NUM; alternate key = ACCT-ID).
  • ACCTDAT VSAM: Account data — retrieved when displaying card details.
  • CUSTDAT VSAM: Customer data — retrieved for customer name display.

📊 Card Cross-Reference Structure

CARDXREF Record:
  FD-XREF-CARD-NUM   PIC X(16)  - Card number (primary key)
  FD-XREF-CUST-NUM   PIC 9(9)   - Customer ID
  FD-XREF-ACCT-ID    PIC 9(11)  - Account ID (alternate key)

📁 Key Files

  • app/cbl/COCRDLIC.cbl — Card list
  • app/cbl/COCRDSLC.cbl — Card detail view
  • app/cbl/COCRDUPC.cbl — Card update
  • app/cpy/CVCRD01Y.cpy — Card work area
  • app/data/ASCII/carddata.txt — Sample card data
  • app/data/ASCII/cardxref.txt — Sample cross-reference

✅ Acceptance Criteria Patterns

Card List:
  • Given an account with multiple cards
  • When user navigates to card list
  • Then all cards for the account are shown with card number, status, and expiration
  • And user can page forward/backward through the list
Card Detail View:
  • Given a valid card number
  • When user selects a card
  • Then all card fields are displayed along with linked account and customer info
Card Update:
  • Given an admin user on the card update screen
  • When card status is changed and confirmed
  • Then CARDDAT is updated and confirmation message shown
