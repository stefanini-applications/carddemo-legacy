📐 Architecture Overview

CardDemo is a CICS pseudo-conversational application. Each CICS program handles one user interaction, sends a BMS screen, and returns control to CICS. Navigation state is carried via the COMMAREA (CARDDEMO-COMMAREA).


The application has two user roles: Regular User (U) — accesses accounts, cards, transactions, bill payment, and reports; Admin (A) — additionally manages users and system configuration.


Core data lives in VSAM KSDS files. Extension modules add DB2, IMS DB, and MQ capabilities for richer integration patterns.
