⚙️ Batch Processing

End-of-Day JCL Pipeline — Transaction Posting, Interest, Statements

📋 Overview

Module ID: batch-processing

Purpose: End-of-day sequential batch pipeline that posts daily transactions, calculates interest, produces statements, and generates reports. All CICS files must be closed before batch runs and reopened afterward.

Business Context: The batch pipeline is the backbone of the financial processing cycle. It runs nightly, processes all daily transactions, updates account balances and category balances, applies interest rates, and produces customer-facing statements.

⚙️ Batch Pipeline (Sequential Order)

CLOSEFIL→ ACCTFILE→ CARDFILE→ XREFFILE→ CUSTFILE→ TRANBKP→ TRANCATG→ TRANTYPE→ DISCGRP→ TCATBALF→ POSTTRAN→ INTCALC→ COMBTRAN→ CREASTMT→ TRANREPT→ OPENFIL

🎯 User Story Templates

As an operations team, I want the batch pipeline to run nightly so that account balances are up to date each morning.
As a batch system, I want POSTTRAN to reject invalid transactions to DALYREJS so that bad records are tracked separately.
As a batch system, I want INTCALC to apply correct interest rates from DISCGRP so that monthly interest charges are accurate.
As an operations team, I want WAITSTEP to pause jobs for a configurable time so that I can control pipeline timing.

📊 Story Complexity

Simple 1-2 pts Medium 3-5 pts Complex 5-8 pts

Simple (1-2 pts)

  • CLOSEFIL/OPENFIL: IEFBR14 DD dummy steps
  • File refresh jobs (ACCTFILE, CARDFILE, CUSTFILE)

Medium (3-5 pts)

  • COMBTRAN: SORT with control cards
  • CBEXPORT/CBIMPORT: data exchange programs
  • CBCUS01C: customer account utility

Complex (5-8 pts)

  • POSTTRAN (CBTRN02C): multi-file transaction posting
  • INTCALC (CBACT04C): interest calculation pipeline
  • CREASTMT (CBSTM03A): statement generation

✅ Acceptance Criteria Patterns

POSTTRAN: Given DALYTRAN sequential file with N records, when CBTRN02C runs, then: valid records written to TRANFILE, ACCTFILE balances updated, TCATBALF updated, invalid records written to DALYREJS with reason code, completion counts written to SYSOUT.
INTCALC: Given TCATBALF has balances and DISCGRP has rates, when CBACT04C runs, then each account's balance is updated: new balance = balance × (1 + rate/12).
File Dependency: CLOSEFIL must complete before any batch job that reads/writes VSAM files; OPENFIL must run after all batch jobs complete.
Reject Handling: DALYREJS must contain all rejected transactions with: original record + rejection reason code; processing must continue after each rejection.
CREASTMT: Given complete transaction file, when CBSTM03A runs, then one statement per account is produced with all period transactions and balance summary.

🔗 Dependencies

  • All VSAM files: Must be closed (CLOSEFIL) before batch accesses them
  • DISCGRP VSAM: Required by INTCALC for interest rate lookup
  • DALYTRAN: Sequential input file for POSTTRAN
  • DALYREJS: Sequential reject output from POSTTRAN
  • asm/MVSWAIT.asm: Called by COBSWAIT for timer functionality
  • asm/COBDATFT.asm: Date format conversion utility

⚡ Development Acceleration

  • CBTRN02C pattern: Multi-file batch template — sequential input, random VSAM update, sequential reject output
  • CSUTLDTC.cbl: Date utility routines reusable across all batch programs
  • CBSTM03A/B: Main program + subprogram pattern for complex batch output
  • JCL proc pattern: Common DD statements in app/proc/ — reuse for new jobs
