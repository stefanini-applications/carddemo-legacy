👥 User Management

User Story Development Guide — CardDemo Application

📋 Overview

Purpose: Admin-only CRUD operations for managing user accounts in the USRSEC VSAM security file. Admins can list, add, update, and delete users.

Business Context: User management controls who can access CardDemo and what they can do. Proper user lifecycle management (onboarding, role changes, offboarding) is essential for security.

Admin Only COUSR00C–COUSR03C VSAM: USRSEC Admin Menu Options 1–4

🎯 User Story Development Guide

📝 Module-Specific US Templates

  • User List: As an admin, I want to view a list of all system users so that I can see who has access to CardDemo.
  • User Add: As an admin, I want to create a new user account so that a new employee can access the system.
  • User Update: As an admin, I want to update a user's type or password so that I can manage access changes.
  • User Delete: As an admin, I want to delete a user account so that departing employees lose system access immediately.
  • Access Control: As the system, I want to prevent non-admin users from accessing user management so that security is enforced.

📊 Complexity Guidelines

ComplexityPointsExamples
Simple1–2List users (pageable), view user details

🔧 Technical Foundation

🏗️ Key Components

  • COUSR00C.cbl: User list — pageable display (10/page) of all USRSEC records; admin selects user for detail.
  • COUSR01C.cbl: User add — create new user record in USRSEC file.
  • COUSR02C.cbl: User update — modify existing user (name, password, type).
  • COUSR03C.cbl: User delete — remove user record from USRSEC.
  • COADM01C.cbl: Admin menu — routes to user management programs (options 1–4).
  • CSUSR01Y.cpy: User security record layout (80 bytes).

🔗 Dependencies

  • USRSEC VSAM: User security file (KSDS, key = SEC-USR-ID 8 chars).
  • Authentication module: User must be type 'A' to access user management; enforced by COADM01C.

📊 User Record Structure

SEC-USER-DATA (80 bytes):
  SEC-USR-ID      PIC X(8)   - User ID (primary key)
  SEC-USR-FNAME   PIC X(20)  - First name
  SEC-USR-LNAME   PIC X(20)  - Last name
  SEC-USR-PWD     PIC X(8)   - Password (plaintext - demo only)
  SEC-USR-TYPE    PIC X(1)   - 'A'=Admin, 'U'=Regular User
  SEC-USR-FILLER  PIC X(23)  - Reserved

📁 Key Files

  • app/cbl/COUSR02C.cbl — User update

✅ Acceptance Criteria Patterns

Access Control:
  • Given a regular user (type 'U') is authenticated
  • When they attempt to access user management (CA00 menu)
  • Then access is denied; user is returned to their regular menu
User Add — Duplicate Check:
  • Given a user ID that already exists in USRSEC
  • When admin attempts to add user with that ID
  • Then duplicate error displayed; record NOT created
User Delete:
  • Given admin confirms user deletion
  • When COUSR03C processes deletion
  • Then user record removed from USRSEC; confirmation shown
  • And deleted user can no longer authenticate
