📋 Module Overview

Purpose: Admin functions for listing, adding, updating, and deleting user accounts in the CardDemo USRSEC security file.

Business Context: User management is an administrative function to control who can access the system and with what role. Admins create accounts for new employees, update credentials, and revoke access for departing users.

Transactions: CU00 (list) | CU01 (add) | CU02 (update) | CU03 (delete)
Programs: COUSR00C / COUSR01C / COUSR02C / COUSR03C

Admin Only COBOL CICS VSAM USRSEC BMS Maps

🎯 User Story Templates

US-USR-001: As an administrator, I want to list all users so that I can review who has access to the system.
US-USR-002: As an administrator, I want to add a new user with a specified role (Regular or Admin) so that I can grant system access to a new employee.
US-USR-003: As an administrator, I want to update a user's password so that I can assist with credential resets.
US-USR-004: As an administrator, I want to delete a user so that I can revoke access for a departing employee.
US-USR-005: As an administrator, I want to change a user's type from Regular to Admin so that I can elevate their privileges.

✅ Acceptance Criteria Patterns

Add User – Success:
GIVEN a unique User ID, first/last name, password, and user type
WHEN CU01 is submitted
THEN a new SEC-USER-DATA record is written to USRSEC and a success message is displayed
Add User – Duplicate:
GIVEN a User ID that already exists in USRSEC
WHEN CU01 is submitted
THEN an error message indicating duplicate User ID is displayed
Delete User – Confirmation:
GIVEN a valid existing User ID
WHEN CU03 is submitted
THEN the USRSEC record is deleted; the user can no longer log in
Access Control:
GIVEN a Regular User attempts to access CU00
WHEN the transaction is invoked
THEN access is denied and the user is redirected to the main menu

Data Model – SEC-USER-DATA (CSUSR01Y)

01 SEC-USER-DATA.
   05 SEC-USR-ID       PIC X(08)   -- User ID (max 8 chars, VSAM key)
   05 SEC-USR-FNAME    PIC X(20)   -- First name
   05 SEC-USR-LNAME    PIC X(20)   -- Last name
   05 SEC-USR-PWD      PIC X(08)   -- Password (max 8 chars, plain text)
   05 SEC-USR-TYPE     PIC X(01)   -- R = Regular User, A = Admin User
   05 SEC-USR-FILLER   PIC X(23)

Business Rules

  • Only Admin users (SEC-USR-TYPE = 'A') can access CU00–CU03
  • User ID is the VSAM key – must be unique across USRSEC
  • User type determines accessible menu: R → Main Menu, A → Admin Menu
  • Passwords stored in plain text in base app; RACF provides production security
  • USRSEC initialized via DUSRSECJ batch job before first use

Dependencies

  • authentication: Admin session required (CA00 → COADM01C entry point)
  • USRSEC VSAM: Loaded/initialized via DUSRSECJ JCL

⚡ Development Acceleration Factors

  • CSUSR01Y copybook: Standard user record – shared with authentication module
  • COUSR00C list pattern: Established list-with-pagination pattern – reuse for other admin list screens
  • Full CRUD pattern: CU00–CU03 provides a complete CICS CRUD reference implementation for VSAM
