🏦 Accounts

User Story Development Guide — CardDemo Mainframe Application

Overview

Provides view (CAVW / COACTVWC) and update (CAUP / COACTUPC) capabilities for credit card account records stored in the ACCTFILE VSAM KSDS (300-byte records). Account-to-card cross-reference is maintained in CARDXREF. Account balances are updated by nightly batch programs (CBTRN02C, CBACT04C).

Technical Foundation

Writing Good Stories for This Module

  • Distinguish clearly between view-only (COACTVWC) and update (COACTUPC) scenarios.
  • Reference ACCT-ACTIVE-STATUS in acceptance criteria; inactive accounts behave differently.
  • Note that balance is updated by batch (nightly); real-time balance reflects last batch cycle.
  • Credit limit changes may have downstream impact on batch interest calculation (CBACT04C).

Complexity Grid

🟢 Simple (1–2 pts)

  • View account details (read-only screen)
  • Display available credit calculation on screen
  • Add a new display-only field to the view map

🟡 Medium (3–5 pts)

  • Update billing fields (address, zip) with format validation
  • Add new updatable field to COACTUPC with VSAM rewrite
  • Status change (active → inactive) with confirmation prompt

🔴 Complex (5–8 pts)

  • Credit limit change with cross-system notification via MQ
  • Account status change with downstream batch impact analysis
  • Account merge or consolidation with cross-reference update

Acceptance Criteria

GivenWhenThen
ACCT-ACTIVE-STATUS = Y for the requested account User views account via CAVW Current balance and credit limit are displayed correctly
An invalid zip code format is entered Account update is submitted via CAUP Error message is displayed and the ACCTFILE record is not updated
Batch job posts new transactions to the account balance User views the account on the following day Balance reflects all transactions posted by the nightly batch
ACCT-ACTIVE-STATUS = N User attempts to update a field on the account Update is rejected with an appropriate error message

Business Rules

  • Balance calculation: Current balance = prior balance + cycle credits − cycle debits; updated by batch (CBTRN02C, CBACT04C).
  • Interest rates: ACCT-GROUP-ID links to DISCGRP disclosure group file for applicable interest rate.
  • Account status: Account must be active (ACCT-ACTIVE-STATUS=Y) for transactions to be authorized and posted.
  • Cross-reference: CARDXREF maps cards to accounts; multiple cards may belong to one account.
  • Record size: ACCTFILE is 300-byte fixed-block KSDS; updates are full-record rewrites.
