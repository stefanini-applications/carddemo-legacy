📦 MQ Account Extractions

Guide for MQ-driven account and system date inquiries that demonstrate asynchronous VSAM extraction patterns.

📋 Overview

Purpose: Expose system date (`CDRD`) and account detail (`CDRA`) lookups over MQ, highlighting request/response correlation with VSAM back-ends.

Business Context: Teams building portals or integrations need deterministic MQ contracts to retrieve host data without direct CICS terminal access.

🎯 User Story Development Guide

📝 Module-Specific US Templates

  • Date service: As a scheduler, I want to send `DATE` requests to `CARDDEMO.REQUEST.QUEUE` and receive the host date so nightly jobs align with the host system time.
  • Account inquiry: As a channel integrator, I want to invoke `CDRA` with an account number and receive the 300-byte VSAM snippet so my mobile app displays current balances.
  • MQ validation: As a platform tester, I want responses to include the original correlation ID so tracing across queues is reliable.

⚡ Development Acceleration Factors

  • MQ copybooks: `DATE-REQUEST-MSG` and `ACCT-REQUEST-MSG` structures are documented in `app/app-vsam-mq/README.md` and can be reused for new queue contracts.
  • CICS connection defs: `MQCONN`, `MQQUEUE` resources are pre-defined, so new queue listeners simply need matching names.
  • VSAM integration: `CODATE01`/`COACCT01` contain the VSAM access logic that can be repurposed for other MQ-triggered data pulls.

📊 Complexity Guidelines

  • Simple (1-2 pts): Add a new field (e.g., customer name) to the `ACCT-RESPONSE-MSG` payload.
  • Medium (3-5 pts): Expand `CDRA` to allow searching by card number in addition to account number with updated validation for PF keys.
  • Complex (6-8 pts): Build a new MQ transaction that aggregates multiple VSAM files, requires new queue definitions, and updates correlation handling.

🔧 Technical Foundation

🏗️ Key Components

  • Transactions `CDRD` (`CODATE01`) and `CDRA` (`COACCT01`).
  • MQ resources: `MQCONN(MQ01)` and queues `CARDDEMO.REQUEST.QUEUE`, `CARDDEMO.RESPONSE.QUEUE`.
  • Message structures defined in the readme (e.g., `ACCT-REQUEST-MSG`, `ACCT-RESPONSE-MSG`).

🔗 Dependencies

  • IBM MQ subsystem with channel definitions accessible from CICS.
  • VSAM datasets for account data and system parameters.
  • CICS MQCONN logic integrated into `CODATE01`/`COACCT01`.

🌐 Public Interfaces


CDRD - System date inquiry via MQ (CODATE01)
CDRA - Account detail inquiry via MQ (COACCT01)
MQ queues: CARDDEMO.REQUEST.QUEUE → CARDDEMO.RESPONSE.QUEUE

✅ Acceptance Criteria Examples

  • Requests carry a correlation ID echoed in responses for traceability.
  • `CDRA` validates the requested account exists in VSAM before building the 300-byte response.
  • `CDRD` always returns the host date even if the request payload is empty.
