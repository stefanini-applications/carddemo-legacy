📦 VSAM + MQ Extractions

Guide for stories that expose VSAM data via MQ request/response patterns.

📋 Overview

Purpose: Demonstrate asynchronous MQ calls (CDRD/CDRA) that query VSAM system date and account data and return structured responses for MQ consumers.

Business Context: Useful when modern services need synchronized host information without running embedded BMS screens, enabling read-only integration.

🎯 User Story Development Guide

📝 Module-Specific US Templates

  • Date sync: As a scheduler, I want CDRD to return system time so my nightly jobs align with the host clock.
  • Account echo: As an integration tester, I want CDRA to deliver account snapshots via MQ so remote ledger systems can reconcile balances.
  • Reliability: As an MQ operator, I want correlation IDs surfaced so retries trace to the original request.

⚡ Development Acceleration Factors

  • Fixed-length message formats already defined in the README for quick reference.
  • Simple COBOL programs (CODATE01, COACCT01) provide templates for parsing/requesting MQ message structures.
  • Existing MQ connection definitions (`DEFINE MQCONN`, `MQQUEUE`) ensure new message types reuse the same channel configuration.

📊 Story Complexity Guidelines

  • Simple (1–2 pts): Add a new field to the CDRD response map (e.g., `HOST-TZ`).
  • Medium (3–5 pts): Extend CDRA to send additional account fields and include validation for missing VSAM records.
  • Complex (5–8 pts): Support encrypted MQ payloads, requiring new parsing logic in `CODATE01`/`COACCT01` plus key management.

✅ Acceptance Criteria Examples

  • CDRD/CDRA respond within 3 seconds and honor correlation IDs in replies.
  • CDRA only returns data when the account number exists in VSAM cross-reference files; otherwise, it returns an error string in the MQ payload.
  • MQ connection definitions (`AWS.M2.CARDDEMO.PAUTH.REQUEST/REPLY`) remain unchanged.

🔧 Technical Foundation

🏗️ Key Components

  • `CODATE01` handles system date requests and formats `DATE-RESPONSE-MSG`.
  • `COACCT01` queries VSAM account records and populates `ACCT-RESPONSE-MSG` with 300-byte data blocks.
  • MQ message definitions in the README double as documentation for request/response structure.

🔗 Dependencies

  • MQ artifacts under `app/app-vsam-mq/csd/` should be installed in the CICS CSD for transactions CDRD/CDRA.
  • VSAM account cross-references (`CARDXREF`) must remain synchronized with `CARDDATA` and `ACCTDATA`.
  • CICS MQCONN definitions must point to the same queue manager used by the authorization module.

🌐 Public Interfaces

CDRD - Inquire system date via MQ (CODATE01)
CDRA - Inquire account details via MQ (COACCT01)
