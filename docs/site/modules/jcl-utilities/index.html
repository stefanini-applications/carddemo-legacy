📦 jcl-utilities

Guide for stories that move data in/out of CardDemo via utility jobs.

📋 Overview

Purpose: Provide auxiliary jobs (FTP, TXT2PDF, DB2/IMS exchange, internal reader) that operators use for maintenance, audit, and integration.

Business Context: These jobs sit outside the core online/batch experience but are essential for modernization demos showing data handoff to external systems.

🎯 User Story Development Guide

📝 Module-Specific US Templates

  • FTP Export: As an Integration Engineer, I want FTPJCL.JCL to copy a dataset to a partner server so that downstream analytics can consume the ledger.
  • Text-to-PDF: As a Compliance Officer, I want TXT2PDF1.JCL to create a PDF-friendly bill for archiving.
  • DB2 Movement: As a DB2 Administrator, I want CBEXPORT.jcl and CBIMPORT.jcl to move table snapshots in/out of CardDemo.

⚡ Development Acceleration Factors

  • Job templates: The `app/jcl/` folder already contains FTP, TXT2PDF, CBIMPORT/CBEXPORT, and INTRDR jobs—duplicate the structure for new utility jobs.
  • Scripts: `scripts/upld_module.sh` and `scripts/local_compile.sh` orchestrate remote job submission.
  • Control definitions: `app/ctl/` control cards can be reused for dataset definitions and sort steps.

📊 Complexity Guidelines

  • Simple (1-2 pts): Adjust a dataset name or FTP destination in `FTPJCL.JCL`.
  • Medium (3-5 pts): Add a new TXT2PDF conversion that accepts extra metadata fields.
  • Complex (5-8 pts): Build a multi-step utility that coordinates IMS load/unload with DB2 staging and notifications.

✅ Acceptance Criteria Examples

  • FTP/TXT2PDF jobs must land their output in the expected HLQ and include proper `DD DISP` parameters.
  • DB2 load/unload jobs (CBIMPORT/CBEXPORT) must include SQLCA handling and end with condition code 0.
  • Internal reader jobs must not run concurrently with active data load jobs to avoid dataset contention.

🔧 Technical Foundation

🏗️ Key Components

  • Utility JCL in app/jcl/: FTPJCL.JCL, TXT2PDF1.JCL, CBIMPORT.jcl, CBEXPORT.jcl, INTRDRJ1.JCL, INTRDRJ2.JCL.
  • Control cards from app/ctl/ filtered by job type.
  • Scripts under scripts/ that standardize HLQ substitution and remote submission.

🔗 Dependencies

  • FTP jobs require an external SFTP/FTP endpoint and credentials stored in the job profile.
  • TXT2PDF and internal reader jobs depend on VSAM datasets produced by core-batch.
  • DB2 load/unload jobs need the DB2 subsystem up and the target tables accessible.

🌐 Public Interfaces

FTPJCL.JCL, TXT2PDF1.JCL, CBIMPORT.jcl, CBEXPORT.jcl, INTRDRJ1.JCL, INTRDRJ2.JCL (internal reader), supporting control cards in app/ctl/
