🔐 Authentication

User Story Development Guide · CardDemo Mainframe Application

Core ModuleCC00 · COSGN00C

📋 Overview

Purpose: Controls user sign-on and access to the CardDemo application. Validates credentials against the USRSEC VSAM security file and routes users to role-appropriate menus.

Business Context: Every CardDemo session begins at the signon screen. Successful authentication routes admin users to the Admin Menu (CA00) and regular users to the Main Menu (CM00). Failed authentication displays an error and stays on the signon screen.

CICS Transaction: CC00 → COSGN00C → COSGN00 BMS mapset

🎯 User Story Development Guide

📝 Module-Specific US Templates

Cardholder sign-in: As a cardholder, I want to sign in with my User ID and password so that I can access my account management functions.
Admin sign-in: As an administrator, I want to sign in with admin credentials so that I can access user management and admin-only functions.
Invalid credentials: As a user, I want to see a clear error message when my credentials are wrong so that I understand why sign-in failed.
Session routing: As a user, I want to be automatically routed to the correct menu after sign-in so I see only relevant functions for my role.

⚡ Development Acceleration Factors

  • COSGN00C: Fully implemented — VSAM read, password compare, and routing logic already present
  • COCOM01Y copybook: COMMAREA structure reusable for passing user identity between all programs
  • CSUSR01Y copybook: User security record layout shared between authentication and user management
  • BMS map (COSGN00): Existing 3270 screen definition with User ID, Password, and error message fields
  • Two user types only: Simple branch on SEC-USR-TYPE field (A/U) — no complex role matrix

🔗 Dependencies

  • USRSEC VSAM file: Must be initialized (DUSRSECJ JCL job) before CC00 will work
  • CICS region: CC00 transaction and COSGN00C program must be defined and installed
  • COMEN01C (CM00): Regular user landing page — authentication transfers control to this program
  • COADM01C (CA00): Admin landing page — authentication transfers control to this program

🌐 CICS Interface

Transaction: CC00
Program:     COSGN00C
Mapset:      COSGN00
Map:         COSGN0A

Flow:
  EIBCALEN = 0  → First entry, display empty signon screen
  ENTER key     → Validate credentials:
                    READ USRSEC WHERE SEC-USR-ID = input user ID
                    IF SEC-USR-PWD = input password THEN
                      IF SEC-USR-TYPE = 'A' → XCTL COADM01C (CA00)
                      ELSE                  → XCTL COMEN01C (CM00)
                    ELSE display error message

📂 User Security Record Layout

01 SEC-USER-DATA.
    05 SEC-USR-ID     PIC X(08)   -- User ID (VSAM primary key)
    05 SEC-USR-FNAME  PIC X(20)   -- First name
    05 SEC-USR-LNAME  PIC X(20)   -- Last name
    05 SEC-USR-PWD    PIC X(08)   -- Password (plain text in base app)
    05 SEC-USR-TYPE   PIC X(01)   -- A=Admin, U=Regular user
    05 SEC-USR-FILLER PIC X(23)   -- Reserved / padding

✅ Acceptance Criteria Patterns

  • GIVEN the USRSEC VSAM file is initialized, WHEN a user enters valid User ID and password, THEN they are routed to the correct menu (CM00 or CA00)
  • WHEN an invalid User ID is entered, THEN an error message is displayed and the signon screen remains
  • WHEN a correct User ID but wrong password is entered, THEN an error message is displayed
  • WHEN sign-in succeeds for admin user (type A), THEN the Admin Menu (CA00) is displayed
  • WHEN sign-in succeeds for regular user (type U), THEN the Main Menu (CM00) is displayed
  • WHEN the CLEAR key is pressed, THEN the signon fields are cleared
