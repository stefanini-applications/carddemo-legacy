📋 Module Overview

Purpose: Offline data processing for transaction posting, interest calculation, statement generation, data loading, and reporting. Runs as z/OS batch jobs using JCL.

Business Context: The batch module handles all overnight and scheduled processing. After online users add transactions during the day, batch jobs post them to accounts, calculate interest, generate statements, and prepare data for the next online cycle.

🎯 User Story Templates

US-BATCH-001: As a batch operator, I want to run POSTTRAN nightly so that daily transactions are posted to account master files.
US-BATCH-002: As a finance team member, I want to run INTCALC monthly so that interest charges are correctly applied to accounts based on balance and group rates.
US-BATCH-003: As a batch operator, I want CLOSEFIL to run before and OPENFIL after data loading so that VSAM files are accessible to CICS users after refresh.
US-BATCH-004: As a finance team, I want COMBTRAN to merge transaction files so that CREASTMT produces a complete statement.
US-BATCH-005: As an operations team, I want the full batch sequence to run in dependency order so that data integrity is maintained end-to-end.

✅ Acceptance Criteria Patterns

POSTTRAN Success:
GIVEN DALYTRAN contains new daily transactions
WHEN POSTTRAN runs CBTRN02C
THEN all records from DALYTRAN are posted to TRANSACT, account balances updated, return code = 0
INTCALC Success:
GIVEN accounts with non-zero balances and DISCGRP rates defined
WHEN INTCALC runs CBACT04C
THEN interest charges are calculated and applied to ACCTFILE, return code = 0
File Refresh Sequence:
GIVEN CLOSEFIL has completed successfully
WHEN ACCTFILE/CARDFILE/CUSTFILE load jobs run
THEN VSAM files contain fresh data; OPENFIL makes them available to CICS

Full Batch Sequence (Production Run Order)

1.  CLOSEFIL  - Close VSAM files in CICS
2.  ACCTFILE  - Reload Account master
3.  CARDFILE  - Reload Card master
4.  CUSTFILE  - Reload Customer data
5.  XREFFILE  - Reload Card/Account cross-reference
6.  TRANBKP   - Backup Transaction database
7.  TRANEXTR  - Extract Tran type/category from DB2 (optional)
8.  TRANCATG  - Load transaction category types
9.  TRANTYPE  - Load transaction type file
10. DISCGRP   - Load disclosure group file
11. TCATBALF  - Load transaction category balance
12. DUSRSECJ  - Setup user security file
13. POSTTRAN  - Post daily transactions (CBTRN02C)
14. INTCALC   - Run interest calculations (CBACT04C)
15. COMBTRAN  - Combine system + daily transactions
16. CREASTMT  - Produce statements (CBSTM03A) to GDG
17. TRANIDX   - Define AIX on transaction file
18. OPENFIL   - Open files in CICS

Assembler Utilities

  • MVSWAIT (ASMWAIT.mac): Provides configurable wait timer for batch jobs – used in WAITSTEP jobs
  • COBDATFT (COCDATFT.mac): Date format conversion between EBCDIC date formats

Dependencies

  • VSAM files must be closed (CLOSEFIL) before any data load batch jobs
  • POSTTRAN depends on DALYTRAN having daily transactions
  • COMBTRAN must complete before CREASTMT
  • DEFGDGB must run before first CREASTMT execution

⚡ Development Acceleration Factors

  • Established batch chain: Sequence is documented and tested – new batch jobs can be inserted into the chain
  • COBDATFT utility: Reusable date conversion; call from any batch program needing date formatting
  • CBTRN02C posting pattern: Sequential VSAM read → process → rewrite pattern; use as template for new posting programs
  • CBACT04C interest pattern: Account-group rate lookup pattern for financial calculations
