📦 transaction-type-db2

Guide for stories that touch the DB2-backed transaction type administration.

📋 Overview

Purpose: Provide administrators with DB2 CRUD operations for transaction types and keep the VSAM reference files synchronized.

Business Context: This extension proves DB2 integration patterns, ensures data consistency, and makes new transaction metadata accessible to the base application.

🎯 User Story Development Guide

📝 Module-Specific US Templates

  • CRUD Basics: As an Admin, I want to add or update a transaction type via CTTU so I can reflect new fees.
  • Cursor Listing: As a Support Person, I want the CTLI screen to page forward/backward through DB2 records for review.
  • Sync Flow: As a Data Engineer, I want to run TRANEXTR after DB2 edits so the VSAM dataset stays in sync.

⚡ Development Acceleration Factors

  • Static SQL templates: Existing copybooks in app/app-transaction-type-db2/dcl/ show SQLCA handling and host variable bindings.
  • Admin menu integration: Options 5 & 6 in CA00 already route to CTLI/CTTU.
  • Batch sync jobs: CREADB21, TRANEXTR, and MNTTRDB2 demonstrate how to move data between DB2 and VSAM.

📊 Complexity Guidelines

  • Simple (1-2 pts): Add a new validation rule inside CTTU for transaction type descriptions.
  • Medium (3-5 pts): Create a new DB2-backed detail screen that joins TRANSACTION_TYPE with TRANSACTION_TYPE_CATEGORY.
  • Complex (5-8 pts): Replace a VSAM lookup with a DB2 query while ensuring TRANEXTR continues to feed the base VSAM file.

✅ Acceptance Criteria Examples

  • CTLI paging uses the existing DB2 cursor with forward/backward support and shows PF7/PF8 hints.
  • Any DB2 SQL change includes SQLCA error handling and returns to the screen if the status code is not zero.
  • Transaction type deletion honors the `DELETE RESTRICT` foreign key so categories remain consistent.

🔧 Technical Foundation

🏗️ Key Components

  • CICS programs COTRTUPC (add/edit) and COTRTLIC (list/detail/delete).
  • BMS maps COTRTUP and COTRTLI with built-in PF navigation.
  • DB2 tables TRANSACTION_TYPE and TRANSACTION_TYPE_CATEGORY defined under app/app-transaction-type-db2/dd/**.
  • Batch jobs CREADB21, TRANEXTR, and MNTTRDB2.

🔗 Dependencies

  • DB2 subsystem must be defined and running (bind plan DB201PLN, DB2 entries DB2TRAN in CICS).
  • Occurs alongside VSAM files so the base transaction catalog is in sync.
  • Admin menu options rely on the `core-online` navigation experience.

🌐 Public Interfaces

CTTU (transaction type add/edit), CTLI (list/edit/delete), CREATEDB21 (create DB2 tables), TRANEXTR (extract to VSAM), MNTTRDB2 (batch maintenance of DB2 records)
