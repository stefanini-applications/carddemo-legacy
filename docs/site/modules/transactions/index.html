📋 Module Overview

Purpose: View, add, and process credit card transactions. Supports listing with pagination, individual transaction view, and new transaction entry.

Business Context: The transaction module is the heart of the credit card management system. Cardholders review their spending; batch jobs post daily transactions to the master file; reports are generated from transaction data.

Transactions: CT00 (list) | CT01 (view) | CT02 (add)
Programs: COTRN00C / COTRN01C / COTRN02C

COBOL CICS VSAM TRANSACT (KSDS+AIX) VSAM DALYTRAN BMS Maps

🎯 User Story Templates

US-TRAN-001: As a cardholder, I want to list my recent transactions so that I can review my spending history.
US-TRAN-002: As a cardholder, I want to view a transaction's details (merchant, amount, date) so that I can identify specific charges.
US-TRAN-003: As a cardholder, I want to add a new transaction so that I can record a purchase that needs to be tracked.
US-TRAN-004: As a cardholder, I want to navigate forward and backward through my transaction list so that I can review older transactions.
US-TRAN-005: As a cardholder, I want new transactions to require valid type and category codes so that data integrity is maintained.

✅ Acceptance Criteria Patterns

Transaction List:
GIVEN an authenticated cardholder
WHEN CT00 is executed
THEN transactions are displayed with ID, type, amount, and merchant name; PF7/PF8 navigate pages
Transaction Add – Success:
GIVEN all required fields are valid (type code, category, amount, merchant, card number)
WHEN CT02 is submitted
THEN the transaction is written to TRANSACT VSAM and a confirmation is displayed
Transaction Add – Validation:
GIVEN an invalid transaction type code is entered
WHEN CT02 is submitted
THEN an error message is displayed and the transaction is not written

Data Model – TRAN-RECORD (CVTRA05Y)

01 TRAN-RECORD.
   05 TRAN-ID               PIC X(16)        -- Transaction ID (16 chars)
   05 TRAN-TYPE-CD          PIC X(02)        -- Transaction type code
   05 TRAN-CAT-CD           PIC 9(04)        -- Category code
   05 TRAN-SOURCE           PIC X(10)        -- Source system
   05 TRAN-DESC             PIC X(100)       -- Description
   05 TRAN-AMT              PIC S9(09)V99    -- Amount (signed, 2 decimals)
   05 TRAN-MERCHANT-ID      PIC 9(09)        -- Merchant identifier
   05 TRAN-MERCHANT-NAME    PIC X(50)        -- Merchant name
   05 TRAN-MERCHANT-CITY    PIC X(50)        -- Merchant city
   05 TRAN-MERCHANT-ZIP     PIC X(10)        -- Merchant ZIP
   05 TRAN-CARD-NUM         PIC X(16)        -- Card number used
   05 TRAN-ORIG-TS          PIC X(26)        -- Original timestamp
   05 TRAN-PROC-TS          PIC X(26)        -- Processing timestamp
   05 FILLER                PIC X(20)

Dependencies

  • authentication: CICS session required
  • credit-cards: TRAN-CARD-NUM must match a valid CARDFILE record
  • transaction-type-db2 (optional): Type codes can come from DB2 via TRANEXTR extraction
  • batch-processing: POSTTRAN job posts DALYTRAN to TRANSACT; INTCALC uses DISCGRP rates
  • reports: CBTRN03C and CBSTM03A read TRANSACT for reporting

⚡ Development Acceleration Factors

  • CVTRA05Y copybook: Standard transaction record – all transaction programs use this structure
  • TRANSACT AIX: Alternate index on TRAN-CARD-NUM enables efficient card-based lookups
  • COTRN00C pagination pattern: PF7/PF8 browse pattern established; reuse for other list screens
  • CT02 validation pattern: Type/category validation against reference VSAM files – reuse pattern for new validations
