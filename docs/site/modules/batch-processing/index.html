⚙️ Batch Processing Module

CardDemo — User Story Development Guide

📋 Overview

Purpose: Core batch infrastructure for data loading, transaction posting, interest calculation, statement generation, and file management. Runs during nightly maintenance windows.

Business Context: Batch processing handles all high-volume, non-real-time operations. CLOSEFIL/OPENFIL bracket the batch window to ensure data integrity. Scheduler integration via CA7 and Control-M definitions.

🎯 User Story Development Guide

📝 US Templates

  • As a batch operator, I want to run the full nightly batch cycle so that all daily transactions are posted and accounts updated
  • As a batch operator, I want CLOSEFIL to run before batch updates so that CICS and batch do not conflict on VSAM files
  • As a DevOps engineer, I want CardDemo batch jobs integrated into Control-M/CA7 so that the schedule is automated
  • As a batch operator, I want WAITSTEP available so that I can add timed pauses in the job stream when needed
  • As a data engineer, I want CBEXPORT/CBIMPORT utilities so that CardDemo data can be migrated to or from the system

⚡ Development Acceleration Factors

  • Scheduler definitions: CA7 and Control-M job definitions in app/scheduler/
  • REPROC proc: Standard IDCAMS REPRO procedure reused across data load jobs
  • MVSWAIT assembler: Timer utility (WAITSTEP) available for paced batch execution
  • COBDATFT assembler: Date format conversion utility callable from COBOL

📊 Complexity Guidelines

  • 1–2 pts Data load jobs (ACCTFILE, CARDFILE, etc.) — IDCAMS REPRO with sample data
  • 3–5 pts POSTTRAN, INTCALC — multi-file processing with business logic
  • 5–8 pts Statement generation (CREASTMT/CBSTM03A), full batch cycle integration, scheduler setup

✅ Acceptance Criteria Examples

  • Given CLOSEFIL runs → CICS is unable to access VSAM files; batch jobs can update them
  • Given POSTTRAN completes → Accepted records in TRANFILE; rejected records in DALYREJS; total = accepted + rejected
  • Given INTCALC completes → All accounts with balance > 0 have interest added per DISCGRP rates
  • Given CREASTMT completes → Each account has both a plain text statement file and an HTML statement file
  • Given OPENFIL runs → CICS can access VSAM files; online transactions resume
  • Given WAITSTEP with N seconds configured → Job stream pauses for N seconds then continues
