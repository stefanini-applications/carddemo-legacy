💵 Bill Payment

User Story Development Guide — CardDemo Mainframe Application

Overview

Provides a single-screen bill payment capability via CICS transaction CB00 / program COBIL00C. The cardholder submits a payment amount that reduces the current cycle debit balance on the account. The program reads the account from ACCTFILE and resolves the account lookup from CARDXREF. Each payment is also recorded as a transaction in the TRANSACT VSAM file.

Writing Good Stories for This Module

  • Specify that payment amount validation must reject zero and negative values.
  • Note that account active status (ACCT-ACTIVE-STATUS=Y) is a prerequisite for payment.
  • Confirmation stories should reference the dual write: both ACCTFILE update and TRANSACT write must succeed.
  • Balance display stories: current balance is from ACCTFILE and reflects the last batch cycle, not real-time.

Complexity Grid

🟢 Simple (1–2 pts)

  • Display current balance and minimum payment due on payment screen
  • Add a payment history display to the bill payment screen
  • Cosmetic change to COBIL00 BMS map layout

🟡 Medium (3–5 pts)

  • Payment with full amount validation and confirmation screen
  • Minimum payment due calculation and enforcement
  • Partial payment with remaining balance display

🔴 Complex (5–8 pts)

  • Scheduled future payment: store payment intent and process via batch
  • Payment reversal workflow: reverse TRANSACT record and adjust ACCTFILE
  • ACH payment integration via MQ with settlement confirmation

Acceptance Criteria

A valid payment is submitted The transaction list is viewed The payment appears as a transaction with a negative TRAN-AMT (credit)

Business Rules

  • Payment amount: Must be positive and non-zero; zero and negative amounts are rejected.
  • Account status: Account must be active (ACCT-ACTIVE-STATUS=Y) to accept a payment.
  • Credit field: Payment increments ACCT-CURR-CYC-CREDIT on the account record in ACCTFILE.
  • Transaction record: Payment is written to TRANSACT VSAM as a negative amount (credit transaction).
  • Balance update: Real-time balance (ACCT-CURR-BAL) is recalculated by the nightly batch; the online screen reflects the last posted balance.
  • Account lookup: CARDXREF is used to navigate from card number to account ID before updating ACCTFILE.
