📦 Transaction Types

Guide for Developing User Stories

📋 Overview

Purpose: Provide DB2-backed metadata management for transaction types and categories, while keeping VSAM-friendly snapshots for performance-critical CICS flows.

Business Context: Enables US teams to shift reference-data editing into DB2, decreasing reliance on embedded VSAM edits and paving the way for planned modernization.

🎯 User Story Development Guide

📝 Module-Specific US Templates

  • Metadata CRUD: As an admin, I want to add a new transaction category through CTU/TU so that downstream billing programs have up-to-date codes.
  • Cursor Navigation: As an auditor, I want CTLI to page with PF7/PF8 so that I can browse dozens of transaction types without hitting dataset limits.
  • Batch Sync: As a batch operator, I want TRANEXTR to rebuild VSAM tables after DB2 edits so that core transactions remain in sync.

⚡ Development Acceleration Factors

  • DB2 static SQL templates: COBOL programs already include SQLCA handling and host variables—reuse the same structures when adding fields.
  • Extraction jobs: `TRANEXTR` outputs VSAM-friendly files; extend the job only when new DB2 fields must be consumed by VSAM programs.
  • Admin menu hooks: Options 5/6 (CTLI/CTTU) self-register when job `CREADB21` runs, reducing manual configuration steps.

📊 Complexity Guidelines

  • Simple (1-2 pts): Update field labels or descriptions on CTLI without touching DB2 DDL.
  • Medium (3-5 pts): Extend the DB2 table by adding a new column and ensure the CICS programs and TRANEXTR job handle it.
  • Complex (5-8 pts): Introduce cascading updates between DB2 transaction type tables and optional modules (authorization or MQ) while keeping VSAM in sync.

🔧 Technical Foundation

🏗️ Key Components

  • `CTLI`/`COTRTLIC`: DB2 cursor-driven listing with PF7/PF8 paging and delete/edit actions.
  • `CTTU`/`COTRTUPC`: Add/edit form for transaction types.
  • `CREADB21`: Initial DB2 object creation script (tables + indexes).
  • `TRANEXTR` + `MNTTRDB2`: Batch jobs to sync VSAM files with DB2.

🔗 Dependencies

  • DB2 subsystem with plans, transactions, and SQLCA-aware COBOL programs.
  • VSAM copies of transaction-type reference data consumed by core flows.
  • Admin menu entries that surface CTLI/CTTU as options 5 and 6 after installation.

🌐 Public Interfaces

CTLI (CICS) - List/update/delete transaction types (forward/backward cursors)
CTTU (CICS) - Add/edit transaction types
CREADB21 (JCL) - Create DB2 tables/translations
TRANEXTR, MNTTRDB2 (JCL) - Sync DB2 rows with VSAM files
COBTUPDT (Batch) - Process bulk transaction-type updates

📌 Acceptance Criteria Examples

  • DB2 SQL statements must check SQLCA before committing; failures trigger rollback and clear error messages.
  • Deletes are blocked when `TRANSACTION_TYPE_CATEGORY` rows reference the code (DELETE RESTRICT). The UI shows the reason using existing error copybooks.
  • TRANEXTR output files match the VSAM structure expected by the core module and are refreshed after each CTLI/CTTU change.
