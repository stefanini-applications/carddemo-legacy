🗄️ Transaction Type DB2

User Story Development Guide — CardDemo Application (Extension Module)

📋 Overview

Purpose: Manage transaction type reference data in IBM DB2. Admins can list, add, update, and delete transaction types and categories using CICS programs with embedded static SQL.

Business Context: Transaction types classify financial events (sales, credits, fees, interest). This module demonstrates DB2 integration patterns including cursor-based pagination, static embedded SQL, and CRUD operations in a CICS environment.

COTRTLIC COTRTUPC TranID: CTLI / CTTU DB2 Static SQL Admin Only

🎯 User Story Development Guide

📝 Module-Specific US Templates

  • List Transaction Types: As an admin, I want to browse transaction types in DB2 so that I can see all defined codes and descriptions.
  • Add Transaction Type: As an admin, I want to add a new transaction type so that new payment categories are available for processing.
  • Update Description: As an admin, I want to update a transaction type description so that it reflects current business terminology.
  • Delete Transaction Type: As an admin, I want to delete an unused transaction type so that the reference data stays clean.
  • Batch Extract: As a batch operator, I want to extract DB2 transaction types to VSAM-compatible files so that the base application can use them.

🔧 Technical Foundation

🏗️ Key Components

  • COTRTLIC.cbl: Transaction type list/update/delete. Uses DB2 cursor for forward/backward paging. TranID: CTLI.
  • COTRTUPC.cbl: Transaction type add/edit. Uses static embedded SQL INSERT/UPDATE. TranID: CTTU.
  • COBTUPDT.cbl: Batch maintenance for transaction type updates.

🔗 Dependencies

  • DB2 Subsystem: Required for this extension module.
  • CARDDEMO.TRANSACTION_TYPE: Primary DB2 table.
  • CARDDEMO.TRANSACTION_TYPE_CATEGORY: Child table with FK constraint.
  • Admin Authentication: Only admin users can access these programs (Admin Menu options 5 and 6).

📊 DB2 Schema

-- Primary table
CREATE TABLE CARDDEMO.TRANSACTION_TYPE (
  TR_TYPE        CHAR(2)        NOT NULL PRIMARY KEY,
  TR_DESCRIPTION VARCHAR(50)
);

-- Category table
CREATE TABLE CARDDEMO.TRANSACTION_TYPE_CATEGORY (
  TRC_TYPE_CODE     CHAR(2)      NOT NULL,
  TRC_TYPE_CATEGORY CHAR(4)      NOT NULL,
  TRC_CAT_DATA      VARCHAR(50),
  PRIMARY KEY (TRC_TYPE_CODE, TRC_TYPE_CATEGORY),
  FOREIGN KEY (TRC_TYPE_CODE) REFERENCES TRANSACTION_TYPE(TR_TYPE)
    ON DELETE RESTRICT
);

✅ Acceptance Criteria Patterns

Add Transaction Type:
  • Given a unique 2-character type code and description entered
  • When admin submits CTTU transaction
  • Then new row inserted in CARDDEMO.TRANSACTION_TYPE; confirmation shown
Delete with Referential Integrity:
  • Given a transaction type that has related category rows
  • When admin attempts to delete the type
  • Then DB2 FK constraint prevents deletion; error message shown
Cursor Paging:
  • Given more transaction types than fit on one screen
  • When user pages forward then backward
  • Then correct records displayed in DB2 order using cursor navigation
