🏦 CardDemo

Mainframe Credit Card Management — User Story Development Guide

COBOL · CICS · VSAM · JCL · RACF | Optional: DB2 · IMS DB · MQ

📋 Application Overview

CardDemo is a comprehensive mainframe credit card management system built in COBOL/CICS. It provides online (CICS) and batch (JCL) capabilities for managing credit card accounts, transactions, bill payments, and reporting. Entry point: CICS transaction CC00.

User Roles: Regular Users (U) and Admin Users (A). Sample credentials: Admin — ADMIN001 / PASSWORD, User — USER0001 / PASSWORD.

📦 Module Index

Core

🔐 Authentication

Signon, credential validation, user-type routing via USRSEC VSAM. Transaction: CC00

 Core

🏦 Accounts

Account view and update, interest calculation. Transactions: CAVW, CAUP

 Core

💳 Credit Cards

Card list, detail, and update. Transactions: CCLI, CCDL, CCUP

 Core

💸 Transactions

Transaction list, view, add, and batch posting. Transactions: CT00–CT02

 Core

💰 Bill Payment

Online bill payment against account balance. Transaction: CB00

 Core

📊 Reports

Online report request; batch statement generation. Transaction: CR00

 Core

👥 User Management

Admin: list, add, update, delete users. Transactions: CU00–CU03

 Core

⚙️ Batch Processing

Data loading, transaction posting, interest calc, statements. JCL jobs.

 Optional – IMS/DB2/MQ

🔑 Authorization

Pending authorization management via IMS, DB2, and MQ. Transactions: CPVS, CPVD

 Optional – DB2

📋 Transaction Type DB2

Manage transaction type reference data in DB2. Transactions: CTTU, CTLI

 Optional – MQ

📡 MQ Integration

Account and date inquiry via MQ request/response. Transactions: CDRA, CDRD
