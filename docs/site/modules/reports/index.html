📊 Reports

User Story Development Guide — CardDemo Mainframe Application

Overview

Online report request screen (CR00 / CORPT00C) submits the TRANREPT batch job via the internal reader (JCL submission). Batch programs CBTRN03C produce a detailed transaction report, while CBSTM03A and CBSTM03B generate account statements in both plain text and HTML formats. Date range parameters are passed via the DATEPARM sequential file.

Technical Foundation

Programs

  • CORPT00C — CICS report request screen: collects parameters and submits JCL via internal reader
  • CBTRN03C — Batch: transaction detail report generator
  • CBSTM03A — Batch: account statement (plain text and HTML)
  • CBSTM03B — Batch: statement formatting subroutine (called by CBSTM03A)

User Story Development Guide

Story Templates

PersonaUser StoryTags

Cardholder As a cardholder, I want to receive a monthly statement in HTML format so that I have a formatted record. Statement
Auditor As an auditor, I want a detailed transaction report so that I can verify all account activity. Audit

Writing Good Stories for This Module

  • Specify whether the story involves online request (CORPT00C) or is a batch-only story.
  • Date range stories must specify the DATEPARM file as the parameter mechanism.
  • Output format stories should distinguish between plain text (STMTFILE) and HTML (HTMLFILE) outputs.
  • Note that CBSTM03A uses mainframe control block addressing, ALTER/GO TO, and 2D arrays — technical debt to note in modernization stories.

Complexity Grid

🟢 Simple (1–2 pts)

  • View transaction list on screen (online, no batch submission)
  • Add a new summary field to the transaction report
  • Change report header text in CBTRN03C

🟡 Medium (3–5 pts)

  • Parameterized date range report submitted via CICS internal reader
  • Add account summary section to CBSTM03A statement output
  • Add new sort/grouping option to transaction report

🔴 Complex (5–8 pts)

  • PDF statement generation from CBSTM03A output
  • Email delivery integration via MQ for statement distribution
  • Multi-account consolidated report across all accounts for a customer

Business Rules

  • Submission mechanism: Report is submitted from CICS via the internal reader (JCL job submission); not run inline.
  • Date filtering: DATEPARM sequential file controls the date range filter used by batch programs.
  • Dual output: Statement is produced in two formats — plain text via STMTFILE and HTML via HTMLFILE.
  • Technical note: CBSTM03A uses mainframe control block addressing, ALTER/GO TO, and 2D arrays — flag as a modernization candidate in technical debt stories.
  • Statement subroutine: CBSTM03B is a formatting subroutine called by CBSTM03A; changes to statement format touch both programs.
