💵 Bill Payment

Process Payments Against Account Balance — CB00 / COBIL00C

📋 Overview

Module ID: bill-payment

Purpose: Allows users to make bill payments against their credit card account. The payment reduces ACCT-CURR-BAL and creates a corresponding payment transaction record in TRANFILE.

Business Context: Monthly bill payments are a core customer activity. The module reads the current account balance, accepts a payment amount, validates it, writes a transaction record, and updates the account balance in a single CICS transaction.

🎯 User Story Templates

As a regular user, I want to view my current balance before paying so that I know the amount owed.
As a regular user, I want to make a payment against my account so that I reduce my outstanding balance.
As a regular user, I want the system to prevent a payment exceeding my balance so that I don't overpay.
As a regular user, I want to see a confirmation of my payment with the updated balance so that I know it was successful.

📊 Story Complexity

Simple 1-2 pts Medium 3-5 pts Complex 5-8 pts

Simple (1-2 pts)

  • Display bill payment screen with current balance
  • Read account balance from ACCTFILE

Medium (3-5 pts)

  • Validate payment amount (positive, ≤ balance)
  • Write payment transaction to TRANFILE
  • Update ACCT-CURR-BAL via REWRITE

Complex (5-8 pts)

  • Scheduled/recurring payment setup
  • Minimum payment calculation
  • Payment reversal workflow

>
Screen Load: Given ACCT-ID in COMMAREA, when CB00 loads, then current ACCT-CURR-BAL is displayed and payment amount field is empty.
Valid Payment: Given valid payment amount ≤ current balance, when ENTER pressed, then: (1) new TRAN-ID generated, (2) payment record written to TRANFILE with type 'PY', (3) ACCT-CURR-BAL reduced by payment amount, (4) success message and updated balance displayed.
Overpayment Validation: Given payment amount > ACCT-CURR-BAL, then error message is displayed and no updates are made to ACCTFILE or TRANFILE.
Zero/Negative Payment: Given payment amount ≤ 0, then validation error is displayed.
PF3: Returns to main menu (CM00) without processing any payment.

Payment Processing Logic

1. READ ACCTFILE (key=ACCT-ID) → display current balance
2. Accept payment amount from user
3. Validate: amount > 0 AND amount <= ACCT-CURR-BAL
4. READ CXACAIX (XREFFILE AIX) for card number
5. GENERATE TRAN-ID (timestamp-based)
6. WRITE TRANFILE (type='PY', amount=payment)
7. REWRITE ACCTFILE (ACCT-CURR-BAL -= payment)
8. Display updated balance + success message

🔗 Dependencies

  • accounts: ACCTFILE must be readable; ACCT-ID in COMMAREA
  • transactions: TRANFILE used to record the payment
  • credit-cards: XREFFILE used to get CARD-NUM for transaction record
  • authentication: Session must have valid user and account context

⚡ Development Acceleration

  • Two-file update pattern: COBIL00C shows atomic TRANFILE write + ACCTFILE rewrite — reuse for any balance-affecting operation
  • FUNCTION CURRENT-DATE: Timestamp generation pattern for TRAN-ORIG-TS
  • STARTBR/READPREV pattern: COBIL00C uses READPREV on TRANFILE to find latest transaction
