🧾 Bill Payment

User Story Development Guide — CardDemo Application

📋 Overview

Purpose: Enable cardholders to pay their credit card balance online. Creates a payment transaction in the TRANSACT file and reduces the account's current balance.

Business Context: Bill payment is the primary way cardholders reduce their outstanding balance. Each payment is recorded as a transaction, updating the account record immediately.

COBIL00C TranID: CB00 VSAM: TRANSACT VSAM: ACCTDAT

🎯 User Story Development Guide

📝 Module-Specific US Templates

  • Pay Balance: As a cardholder, I want to pay my credit card balance online so that I can reduce my outstanding debt and avoid interest charges.
  • View Balance Before Payment: As a cardholder, I want to see my current balance before paying so that I can decide how much to pay.
  • Payment Confirmation: As a cardholder, I want to see a confirmation after payment so that I know my payment was processed.
  • Payment Validation: As the system, I want to validate payment amount so that partial or full payments are accepted but overpayments are rejected.

🔧 Technical Foundation

🏗️ Key Components

  • COBIL00C.cbl: Bill payment program. Reads ACCTDAT via CXACAIX alternate index, creates payment transaction in TRANSACT, updates account balance. TranID: CB00.
  • COBIL00.bms: Bill payment BMS screen.

🔗 Dependencies

  • TRANSACT VSAM: Payment transaction written here.
  • ACCTDAT VSAM: Account balance reduced by payment amount.
  • CXACAIX: Alternate index on ACCTDAT — used to look up account by card number or cross-reference.

📊 Payment Processing Logic

1. Display current balance from ACCTDAT
2. User enters payment amount
3. Validate: amount > 0, amount <= current balance (optional)
4. Create TRAN-RECORD:
     TRAN-SOURCE = 'ONLINE'
     TRAN-TYPE-CD = payment type code
     TRAN-AMT = payment amount (negative = credit)
5. Write to TRANSACT file
6. Update ACCTDAT: ACCT-CURR-BAL = ACCT-CURR-BAL - payment
7. Display confirmation message

✅ Acceptance Criteria Patterns

Successful Payment:
  • Given a valid account with a positive balance
  • When cardholder submits a valid payment amount
  • Then payment transaction created in TRANSACT and account balance reduced
  • And confirmation message displayed with new balance
Invalid Payment Amount:
  • Given payment amount is zero or negative
  • When cardholder submits payment
  • Then validation error displayed; no transaction created
